### 阅读前先认识 12 个最小概念

| 概念 | 最简解释 |
|---|---|
| 股票 / Ticker | 股票是公司股权的一部分；ticker 是交易代码，必须连同公司和交易所确认 |
| 市值 | 股价乘以当前发行在外股数，是市场对股权的定价 |
| 收入 | 公司按会计规则确认的销售或服务金额，不等于合同总额 |
| 利润 | 收入扣除相应成本和费用后的剩余，毛利、营业利润和净利润不是同一层 |
| 现金流 | 真实现金的流入和流出，利润增长不保证现金增加 |
| CapEx | 购买设备、厂房和数据中心等长期资产的资本支出 |
| 稀释 | 新股、股票薪酬、权证或可转债使原股东所占比例下降 |
| Thesis | 一条可验证、可证伪的投资因果链，不是看多口号 |
| 资本流 | 钱从付款人流向云、设备、芯片、网络、存储、电力和材料的路径 |
| 架构 | 一个计算或通信系统如何组合芯片、存储、网络和软件 |
| 瓶颈 | 难扩、难换、会限制整个系统交付的环节 |
| 催化 | 在具体时间可能改变基本面或市场认识的可验证事件 |

阅读时还会反复看到六个“研究符号”，先记白话意思即可：

| 符号 | 白话解释 |
|---|---|
| Claim | 一个需要证据支持的具体主张，例如“某产品已经量产” |
| `UNKNOWN` | 公开证据不足；它是诚实结论，不是待你猜一个数字的空格 |
| GAAP / IFRS | 美国 / 国际财务报告的正式会计规则；管理层调整口径不能替代它们 |
| BOM | Bill of Materials，物料清单；本教程也用它表示从系统向上游拆解的产品链 |
| TTM | Trailing Twelve Months，过去十二个月；它是历史窗口，不自动代表未来产能或收入 |
| Hard Gate | 一票否决检查；关键项过不了，即使主题很热或评分很高也先停止 |

## 0. 你将学会什么

学完本教程，你应当能够独立完成以下任务：

1. 画出一张不混淆层级的 AI 基础设施产业链图。
2. 从大型云厂商的资本开支，追踪资金如何传导到芯片、存储、网络、光互联、先进封装、电力和上游材料。
3. 区分“行业需求强”“供应链有瓶颈”和“某只股票值得研究”这三件不同的事。
4. 分清当前架构、过渡架构和下一代架构，避免把本代赢家永久外推。
5. 用订单、认证、产能、价格、交付周期和客户交叉验证一个瓶颈是否真实。
6. 读懂 10-K、10-Q、8-K、S-1、20-F、6-K 等关键文件的研究用途。
7. 区分 backlog、LOI、MOU、已签合同、take-or-pay 和可确认收入。
8. 用 GAAP 毛利率、SBC、折旧、利息、资本开支和自由现金流检查财务质量。
9. 区分期末股数、GAAP 稀释 EPS 股数和点时稀释情景，并估算 ATM overhang、企业价值和情景估值。
10. 比较纯标的、大公司低纯度业务、海外标的、ETF 和篮子哪一种表达更合适。
11. 写出明确的催化、证伪条件、退出条件和组合相关性。
12. 完成一份可复核、可更新、不会把推断冒充事实的单票研究报告。
13. 建立一张带报告期、抓取日、质量状态、许可范围和 fallback 状态的产业链数据快照。
14. 用同一口径比较 SNDK、MU、Samsung、SK hynix、LITE、COHR、AAOI、AXTI、SIVE 与 SOI，并知道哪些数字必须保留为 unknown。

如果你最后只能记住一句话，请记住：

> 先看谁在花钱，再看钱流到哪里；先证明哪里真的卡，再比较谁能把这个卡点变成普通股股东的回报。

## 1. 先建立正确预期

### 1.1 研究不是预测明天涨跌

AI 产业链研究的目标不是精确预测下一根 K 线，而是回答四个层次的问题：

| 层次 | 问题 | 典型错误 |
|---|---|---|
| 产业 | 哪一类需求正在增长？ | 看到“AI”两个字就认为需求确定 |
| 供应链 | 增长会在哪个环节形成约束？ | 只看下游热门公司，不追上游 |
| 公司 | 哪家公司真正拥有相关收入和利润暴露？ | 把合作、展示、概念归类当作订单 |
| 股票 | 当前价格和证券结构是否还允许股东受益？ | 好公司等于好股票 |

研究做得好，仍然可能暂时亏钱；研究做得差，也可能因为市场情绪暂时赚钱。你需要评价的是过程质量，而不是用一次涨跌倒推研究是否正确。

对初学者，可以把整本教程压缩成必须依次通过的三道门：

```text
第一道门：产业是否真实受益？
    ↓
第二道门：公司是否真正获得收入、利润和现金流？
    ↓
第三道门：这些价值是否能留给当前普通股股东？
```

只回答第一道门，得到的是产业故事；三道门都通过，才接近一条可以继续研究的股票 thesis。

### 1.2 Serenity 框架不等于“不看财务和估值”

这个框架只是改变研究的起点：

```text
传统起点：公司看起来便宜 -> 研究公司为什么便宜
Serenity 起点：资本流和架构变化 -> 找瓶颈 -> 找表达 -> 再判断价格是否合理
```

最终仍然必须回到收入、利润、现金流、股本、债务和估值。供应链故事如果无法传导到股东经济价值，就不是完整的股票 thesis。

### 1.3 小白的安全边界

可以把完成至少十份结构完整的普通股研究报告当作最低训练里程碑，但这不是使用高风险工具的“资格证”。初学阶段建议把以下内容始终放在纸面练习和风险教育中：

- 杠杆和融资买股；
- 裸卖期权；
- 深度虚值短期期权；
- 低流通盘 squeeze；
- IPO、SPAC、权证和解禁事件交易；
- 时区套利；
- 因社交媒体帖子进行当天跟单。

[Investor.gov 的投资与期权教育资料](https://www.investor.gov/introduction-investing/general-resources/news-alerts/alerts-bulletins/investor-bulletins-63)提醒，所有投资都有风险；期权买方可能损失全部权利金，部分期权卖方策略甚至可能面临超过初始投入的损失。初学阶段最重要的是保住学习能力，而不是最大化一次交易的弹性。

### 1.4 不要用公开业绩截图替代方法验证

Serenity 的公开收益和仓位陈述属于自报信息，不是经独立审计的完整账户记录。公开内容还天然存在幸存者偏差、选择性展示和仓位大小未知等问题。

因此，本教程的权重顺序是：

```text
可复核的方法和公开证据
> 可验证的历史 thesis
> 单次交易结果
> 自报收益截图
```

你应复制的是供应链映射、客户验证、财务过滤和证伪纪律，而不是别人的杠杆、仓位或期权合约。

### 1.5 哪些来自原方法论，哪些是教程扩展

为避免把教学设计误认为 Serenity 的原话，来源边界如下：

| 内容 | 来源性质 |
|---|---|
| F-A-B-E-S-R-P 七层框架 | 原方法论核心 |
| 四个策略栈、15 条原则、信号类型、100 分表 | 原方法论整理 |
| 5-60 个交易日 | 原方法论对历史公开帖子的回看性描述 |
| L1-L5 来源等级、claim 状态、质量/许可/fallback 字段 | 本教程结合 Atlas 发布安全增加 |
| 稀释、合同收入/价值、FCF 等启发式公式 | 本教程为操作化增加，非统一会计标准 |
| 虚构案例、hard gate、12 周课程和模板 | 本教程教学设计 |

任何教程扩展都不能覆盖监管文件、会计准则和最新公司事实。

> **公开信息边界：** 本教程没有未公开重大信息、公司内部文件或传统意义上的内幕消息。所谓“行业隐性知识”，只指公开资料中容易被非专业读者误读的口径、流程和产业层级。研究优势来自合法公开信息的正确拼接，而不是获得私有信息。

## 2. AI 基础设施产业链地图与行业词典

### 2.1 画图前先认识五组行业角色

先理解角色，再看缩写。产业链上的同一家公司可能同时属于多组，但每一个收入来源都必须落到具体产品和客户。

| 角色组 | 必须先懂的词 | 白话解释 |
|---|---|---|
| 付款与云 | Hyperscaler / Neocloud / Sovereign cloud | 超大云平台 / 以 GPU 算力为核心的新型云厂商 / 强调本地控制和数据主权的云 |
| 云服务分类 | Cloud infrastructure / IaaS / PaaS / Hosted private cloud | 云基础设施总称 / 基础设施服务 / 平台服务 / 托管私有云；统计市场份额时必须写清范围 |
| 计算 | GPU / ASIC / TPU / CPU / DPU / NIC | 通用并行加速器 / 专用芯片 / 张量处理器 / 通用处理器 / 数据处理卸载 / 网络接口 |
| 存储与制造 | HBM / DRAM / NAND / SSD / Foundry / OSAT / CoWoS | 高带宽内存 / 动态内存 / 闪存 / 固态盘 / 晶圆代工 / 外包封装测试 / 先进封装平台 |
| 网络与光 | DSP / EML / DFB / CW laser / SiPh / CPO | 信号处理 / 激光器相关器件 / 硅光平台 / 共封装光学；器件、光引擎和完整模块不能混为一谈 |
| 上游材料 | Feedstock / Substrate / Epiwafer / InP / SOI | 原料 / 衬底 / 外延片 / 磷化铟材料体系 / 绝缘体上硅平台，代表不同产业层级 |

还要先记住三组容易误读的关系：

- **Scale-up / Scale-out**：前者是在一个紧耦合系统内部扩展，后者是在多个系统或机架之间横向扩展。
- **800G / 1.6T**：通常表示模块聚合速率，不是单通道速率；必须继续检查 lane 数量和每通道速率。
- **HBM 与 DRAM**：HBM 是 DRAM 的子类别，因此 HBM 份额不能与 DRAM 总份额相加。

云公司还常用 **Run-rate** 把当前业务速度年化，它不是已经实现的全年收入；**GPU utilization** 是 GPU 实际被使用的比例，会直接影响收入、毛利和设备回收期。

### 2.2 从需求到物理基础设施

先把 AI 看成一条资本流，而不是一串 ticker：

```text
终端需求
├── 消费者与企业 AI 应用
├── 搜索、广告、推荐、办公和代码工具
└── 模型训练、推理与数据处理
        ↓
算力购买者
├── Hyperscaler：大型云与互联网平台
├── Neocloud：以 GPU/AI 算力租赁为核心的新型云厂商
├── 主权云、政府、科研机构
└── 大型企业与模型公司
        ↓
计算平台
├── GPU
├── ASIC / TPU / 自研加速器
├── CPU / DPU / NIC
└── 服务器、机柜与系统集成
        ↓
关键配套
├── HBM、DRAM、NAND、SSD
├── 交换芯片、DSP、网络接口
├── 铜互联、光模块、激光器、硅光、CPO
├── 晶圆代工、先进封装、基板、测试和量测
└── 电源、变压器、电网、制冷和数据中心
        ↓
上游基础
├── 衬底、外延片、化学品、特种材料
├── 晶圆制造和封装设备
├── 电工钢、铜、光学材料
└── 电力、土地、许可、工程能力和融资
```

任何一层都可能出现机会，但只有满足“需求真实、供给受限、替代困难、股票可表达”时，才值得进入下一步。

### 2.3 每一层靠什么赚钱

| 层级 | 收入来源 | 领先指标 | 主要风险 |
|---|---|---|---|
| 云厂商 | 算力租赁、云服务、软件服务 | 合同、利用率、预订容量、客户信用 | 债务、GPU 折旧、价格竞争、稀释 |
| 芯片 | 加速器、交换和定制芯片出货 | 晶圆订单、封装产能、客户平台节奏 | 架构替代、客户自研、出口限制 |
| 存储 | HBM、DRAM、NAND、SSD 量价 | 合同价、现货价、库存、产能转换 | 周期反转、扩产过快、技术良率 |
| 网络 | 交换机、NIC、DSP、线缆 | 端口速率、集群规模、设计周期 | 标准变化、客户集中、价格压力 |
| 光互联 | 光模块、激光器、光引擎 | 800G/1.6T 放量、认证、lead time | 代际切换、良率、竞争和降价 |
| 材料/衬底 | 晶圆、衬底、外延和特种材料 | 客户认证、扩产周期、利用率 | 替代材料、双供、单一客户 |
| 先进封装 | 代工、封装、测试、量测 | CoWoS/HBM 产能、设备订单、HVM | 巨额 capex、客户自建、周期 |
| 电力电网 | 设备、工程、电力容量 | backlog、交付周期、并网许可 | 审批、利率、原材料和执行 |

### 2.4 初学者最容易混淆的光互联层级

以下角色不能混为一谈：

```text
原料 / feedstock
    ↓
衬底 / substrate
    ↓
外延片 / epiwafer
    ↓
晶圆制造 / foundry
    ↓
激光器、探测器、调制器等器件
    ↓
光引擎 / optical engine
    ↓
光收发模块 / transceiver module
    ↓
交换机、服务器与 AI 集群
```

一家做衬底的公司，不等于一家做外延片的公司；foundry 不等于光模块厂；激光器也不等于完整 transceiver。层级混淆会直接导致错误的市场份额、TAM 和竞争格局判断。

### 2.5 三种技术阶段必须分开

| 阶段 | 含义 | 你要找的证据 |
|---|---|---|
| 当前代际 | 已经贡献真实收入的主流架构 | 出货、收入、毛利、订单、利用率 |
| 过渡代际 | 新旧系统并存，开始认证或局部放量 | design win、qualification、客户平台支持 |
| 下一代 | 潜在 TAM 大，但量产和商业模式未完全验证 | 样品、工程进展、foundry/客户合作、HVM 时间表 |

“下一代技术很重要”与“某公司明年能因此产生高利润”之间，至少隔着认证、良率、量产、客户采购和收入确认五道门。

### 2.6 两段技术历史：为什么标准、封装和收入不会同时发生

#### 2.6.1 从 400G、800G 到 1.6T 与 CPO

IEEE 先建立 200G/400G Ethernet 标准，随后在 IEEE 802.3df-2024 中加入 800Gb/s MAC 和物理层；1.6Tb/s 标准继续推进。OIF 同期推动 co-packaging 共识，并在 2022 年发布 CPO 框架。这个过程形成了今天光互联研究最重要的时间边界：**标准完成、产品展示、客户认证、HVM 和财务收入是五个不同状态**，可插拔模块、线性光学、光引擎和 CPO 也可能长期并行。来源：[IEEE 802.3df](https://standards.ieee.org/ieee/802.3df/11107/)、[OIF Co-Packaging Framework](https://www.oiforum.com/oif-releases-co-packaging-framework-implementation-agreement/)。

> **行业隐性知识｜标准不是订单：** 一项技术进入行业标准，只能证明互操作规则更清楚，不能证明某家公司已经通过特定客户认证、更不能证明当季收入。研究 LITE、COHR、AAOI 或 SIVE 时，必须继续沿“产品—认证—量产—收入—现金”逐级验证。

#### 2.6.2 从 2012 年 CoWoS 测试载具到 AI 系统瓶颈

TSMC 在 2012 年公布首个结合 Wide I/O DRAM 的 CoWoS 测试载具，把逻辑芯片和 DRAM 通过中介层及先进封装集成。到了 AI 加速器时代，逻辑芯片、HBM、中介层、基板、封装、测试和良率共同决定最终系统交付，CoWoS 才从一个后段工艺名称演变为可能约束 GPU 出货的系统层。来源：[TSMC 2012 年 CoWoS 公告](https://pr.tsmc.com/english/news/1746)。

> **行业隐性知识｜CoWoS 不是一颗器件：** 它是由多种器件、材料、工艺和生态伙伴共同完成的集成平台，不能与 HBM 或 GPU 当成三个可相加的市场。

### 2.7 数据中心从“有土地”到“产生收入”

数据中心和 Neocloud 的宣传最容易把不同状态压成一个“MW”数字。正确的投运链是：

```text
Land / site secured
-> Power secured / contracted
-> Energized
-> IT installed
-> Commissioned / available
-> Revenue-generating
```

| 状态 | 白话解释 | 不能自动推导什么 |
|---|---|---|
| Land / site secured | 已取得土地或场址控制 | 已取得电力或已经开工 |
| Power secured / contracted | 公司称已落实或签约电力 | 已并网、已通电或可以立即使用 |
| Energized | 电力已经送达场址 | GPU、服务器和网络已经安装 |
| IT installed | 计算和网络设备已经安装 | 已完成调试或客户已经使用 |
| Commissioned / available | 系统通过测试并可以提供服务 | 已达到目标利用率或毛利 |
| Revenue-generating | 已满足收入确认条件并开始产生收入 | 已产生正自由现金流 |

**GW / MW** 是吉瓦和兆瓦，表示电力容量，不是收入单位。**Active power** 往往是公司自定义 KPI；没有清楚定义时，不能自动把它当作 energized、commissioned 或 revenue-generating capacity。

> **行业隐性知识：** 一份长期合同、一个 1GW 场址或一批 GPU 都可能需要巨额前置资本。研究 Neocloud 时，必须把客户合同、设备折旧、利用率、利息、建设 CapEx 和再融资放在同一张现金流表中。

> **本章产物：** 画一张自己的 AI 基础设施产业链图，并把计算、存储、网络与光、电力与散热、材料、建设投运六层各填入至少一个“产品—公司—验证指标”。

建议采用三遍法：

1. 第一遍只看“白话解释”，知道它大致在说什么；
2. 第二遍结合对应章节，理解它会怎样改变 thesis；
3. 第三遍在自己的研究模板中真正计算或引用一次。

### 3.1 股票和估值

| 术语 | 白话解释 | 研究时怎么用 |
|---|---|---|
| 股价 | 一股股票的交易价格 | 不能单独说明公司贵或便宜 |
| Current shares outstanding | 某一时点实际已发行在外股数 | 与同一时点股价配合计算常规市值 |
| Weighted-average basic shares | 一个报告期内的基本加权平均股数 | 用于 basic EPS，不应直接替代期末股数算市值 |
| GAAP diluted weighted-average shares | 按会计规则纳入当期有稀释作用工具后的加权平均股数 | 用于 diluted EPS；反稀释工具会被排除 |
| Scenario fully diluted shares | 研究者按股价、归属、行权和转换条件建立的点时稀释情景 | 不是统一 GAAP 指标，必须披露假设 |
| 市值 | 股价 × 当前已发行在外股数 | 市场对股权的定价，不等于企业全部价值 |
| 企业价值 EV | 市值 + 债务等 - 现金 | 比较资本结构不同的企业时更实用 |
| P/S | 市值 ÷ 收入 | 常用于尚未稳定盈利的公司，但不代表利润质量 |
| P/E | 市值 ÷ 归属于普通股股东的净利润 | 亏损或利润接近零时通常没有可比意义 |
| FCF yield | 一致口径的股权自由现金流 ÷ 市值 | FCF 不是统一 GAAP 指标；不能把 unlevered FCF 随意除以市值 |
| TAM / SAM | 理论总市场 / 公司产品真正可服务的市场 | TAM 只能说明池子多大；SAM 还要扣除规格、地域、政策、认证和产能限制 |
| 估值重估 | 市场愿意给更高或更低倍数 | 需要新事实、可信预期或风险变化触发 |

### 3.2 财务报表

| 术语 | 白话解释 | 常见陷阱 |
|---|---|---|
| Revenue | 公司确认的收入 | backlog 不等于收入 |
| Gross profit | 收入减直接成本 | 单看金额不如结合毛利率和产品结构 |
| GAAP gross margin | 按会计准则计算的整体毛利率 | 不能拿单业务线或调整后毛利直接比较 |
| Operating expense | 研发、销售、管理等费用 | 高增长公司也必须说明费用能否形成回报 |
| Operating income | 毛利扣除营业费用后的营业利润 | 还没有扣除通常列在营业利润以下的利息和税项 |
| Net income | 扣除利息、税项及其他损益后的净利润 | 必须确认是否归属于普通股股东，以及是否受一次性项目影响 |
| SBC | 用股票支付员工薪酬 | 是 GAAP 费用，并可能造成净股数稀释；回购可抵消股数但会消耗现金 |
| Depreciation | 设备资产分期计入成本 | GPU、设备和数据中心会真实贬值，不能忽略 |
| PPE | Property, Plant and Equipment，物业、厂房及设备 | 报表中的 PPE 购置常用作 CapEx 依据，但租赁、合资和补贴可能另有口径 |
| Interest expense | 债务利息 | 高 capex 公司可能收入增长但股东拿不到利润 |
| CFO | 经营活动现金流 | 会受营运资本变化影响 |
| Working capital | 库存、应收、应付等短期经营项目 | 利润不变时也可能大量占用或释放现金 |
| Capex | 购买设备、数据中心等长期资产的支出 | 增长需要多少资本是股东回报关键 |
| FCF | 常见教学近似是 CFO - capex | 不是统一 GAAP 指标；公司定义、利息分类和租赁处理可能不同 |
| Adjusted EBITDA | 剔除利息、税、折旧摊销及公司自选调整项后的指标 | 不能替代 GAAP 利润和现金流，必须查看调整明细 |
| N/M | Not Meaningful，无可比意义 | 亏损或分母接近零时，不应硬算 P/E、增长率或覆盖倍数 |

### 3.3 其余专业词不要提前硬背

本章只要求先懂股票、三张报表和最基本的估值语言。后面的专业词会在真正改变判断时解释：

| 以后遇到的词 | 先记住什么 | 唯一完整解释位置 |
|---|---|---|
| LOI、MOU、Design win、Qualification、Backlog、RPO、ARR、HVM | 合作、认证、合同、量产和收入不是同一步 | 第 14 章 |
| ATM、可转债、权证、RSU、Lockup、Float、Fully diluted | 好业务可能被债务和新增股数稀释 | 第 10 章 |
| ADR、ADS、主上市、第二上市、OTC、ISIN | 同一公司可能有不同市场、股类和换算比例 | 第 9.5 节与第 20 章 |
| IV、Vega、Theta、Delta、LEAPS、Covered call | 期权是独立高风险工具，不是普通股研究的捷径 | 第 24.1 节 |

建议做法不是背缩写，而是第一次在公告里见到时，立刻回到对应章节回答：它处于哪一步、能证明什么、还不能证明什么。

> **本章产物：** 选一家公司，做一张只含“证券身份、收入、利润、现金流、当前股数、市值”的六格基础卡；其他复杂词先留空。

## 4. Serenity 的总框架：F-A-B-E-S-R-P

七个字母是一条研究管线：

```text
Funnel
  ↓ 谁在花钱，钱流向哪里
Architecture
  ↓ 哪一代架构在放量，哪一代会替代
Bottleneck
  ↓ 什么真正稀缺、难扩、难替代
Exposure
  ↓ 哪个公开市场工具暴露最纯
Security Structure
  ↓ 股本、债务和融资会不会吞掉回报
Repricing
  ↓ 市场为何尚未定价，什么会让它发现
Portfolio
  ↓ 用什么仓位类型表达，错了如何退出
```

注意：原方法论正文先讲重估再讲证券结构，但实际操作应把 `S` 当成进入催化和仓位讨论前的硬过滤器。否则你可能先爱上故事，最后才发现普通股股东根本拿不到收益。

### 4.1 四个策略栈

完整方法论不只有“找半导体瓶颈”，而是四套策略叠加：

| 策略栈 | 研究目标 | 初学者定位 |
|---|---|---|
| 核心瓶颈长仓 | 找多年结构性供需错配 | 重点学习 |
| 全球纯标的 | 从美国需求反推海外供应商 | 学习研究，暂缓低流动性交易 |
| 事件/证券结构 | 研究 IPO、锁定期、权证、融资、诉讼等 | 高级模块 |
| 相对错价/波动率 | 研究 ETF、IV、LEAPS、跨市场滞后 | 高级模块 |

前两个适合训练产业研究；后两个对时点、流动性和工具理解要求更高，不能因为方法论包含它们就认为初学者也应立即使用。

### 4.2 与传统半导体周期框架的关系

两套方法回答不同问题：

| 框架 | 重点变量 | 更擅长回答 |
|---|---|---|
| 传统半导体周期 | 库存、价格、供需、产能、资本开支、竞争格局 | 子行业处于周期哪一段，盈利 beta 如何变化 |
| Serenity 瓶颈框架 | 资本流、架构迁移、多跳 BOM、纯度、证券结构、重估催化 | 哪个未来卡点可能尚未被市场充分定价 |

传统周期分析不能替代供应链 mapping；瓶颈故事也不能跳过库存、价格、产能和资本纪律。两者应叠加使用。

## 5. 证据系统：把“听说”变成可审计结论

Serenity 框架的优势不是获得私有信息，而是更快、更准确地拼接公开信息。因此，证据管理是整个方法的地基。

### 5.1 来源等级

| 等级 | 来源 | 例子 | 使用方式 |
|---|---|---|---|
| L1 | 监管文件和正式合同披露 | SEC filing、政府数据、法院文件 | 事实主锚 |
| L2 | 公司第一方披露 | IR、财报、电话会、产品文档 | 重要，但需防管理层偏差 |
| L3 | 交易对手交叉验证 | 客户、供应商、竞争者披露 | 验证供应链关系和需求 |
| L4 | 有来源的行业资料 | 产业媒体、协会、许可数据 | 补充背景，不覆盖第一方反证 |
| L5 | 评论和社交媒体信息流 | 分析师观点、X、论坛、视频 | 发现线索，不直接当事实 |

来源等级高，不代表结论自动正确。例如监管文件可以验证“公司作出了某项指引”，不能验证“未来结果一定实现”。关键是把“披露行为”“已发生事实”“管理层预期”和“研究者推断”拆成不同 claim。

### 5.2 声明状态

每一个关键 claim 都标记状态：

| 状态 | 定义 | 示例 |
|---|---|---|
| `VERIFIED` | 来源直接支持一个被窄化表述的已发生或已披露事实 | filing 确实披露某项债务条款；不代表未来预测会实现 |
| `CROSS_CHECKED` | 两个独立链条相互印证 | 客户与供应商都确认认证 |
| `SUPPORTED` | 有证据支持，但仍缺关键环节 | 管理层称明年量产，客户未确认 |
| `INFERRED` | 由 BOM、技术或时间关系推断 | 某上游可能受益于下游扩产 |
| `UNKNOWN` | 信息不足 | 未披露客户或业务纯度 |
| `CONTRADICTED` | 存在强反证 | 公司声称短缺，但竞争者宣布快速扩产 |

### 5.3 时效和质量标签

对所有关键数字和事实保留：

```text
Source：来源名称和链接
Observation time：事实对应的观察期
Fetch time：你获取资料的时间
Quality：official / reported / estimated / proxy
Licence：公开展示、元数据、允许摘要或仅内部研究
Fallback：是否使用缓存、替代源或旧快照
Freshness：current / stale / historical
```

如果数据源失败，不要用 demo 数字填空。保留上一版完整结论并明确标记 stale，比制造“看起来完整”的错误数据更安全。

**数据血缘（data lineage）**是一个数字从原始来源、口径、观察期、计算过程到最终展示的完整路径。没有血缘的“精确数字”通常比带有 `UNKNOWN` 的诚实结论更危险。

### 5.4 数字的来源状态

| 状态 | 表示什么 | 使用边界 |
|---|---|---|
| `company_reported` | 公司已经报告的历史事实 | 仍需检查会计口径和报告期 |
| `company_guidance` | 管理层对未来的指引 | 是公司预期，不是已实现事实 |
| `company_reported_non_gaap_kpi` | 公司自定义的非 GAAP 经营指标 | 必须保存定义并检查是否变化 |
| `first_party_calculation` | 研究者基于公开原始数字自行计算 | 必须公开公式、单位和分母 |
| `third_party_estimate` | 独立研究机构或行业资料估算 | 不能冒充官方事实，需检查许可范围 |
| `issuer_estimate` | 发行人对市场规模或自身份额的估计 | 存在宣传偏差，需要独立交叉验证 |
| `unknown` | 缺少足够公开证据 | 是有效结果，不得用猜测填空 |

### 5.5 市场份额也是一个 Claim

写任何市场份额前，先写完整分母：

```text
地域：全球、美国、中国还是某个客户？
产品：DRAM、HBM、NAND、AI accelerator 还是完整模块？
期间：季度、年度还是某个预测年份？
计量：收入、出货量、bit shipment、端口、容量还是装机？
来源：公司估算、第三方估算还是第一方计算？
```

三个典型陷阱：

- HBM 是 DRAM 的子类别，HBM 份额不能与 DRAM 总份额相加；
- Revenue share 与 bit shipment share 可能给出不同排名；
- AI accelerator 的份额取决于分母是否包含 GPU、TPU 和其他 ASIC。

市场份额没有完整分母时，正确状态是 `unknown`，而不是从图表像素、市场规模或公司收入倒推出一个看似精确的百分比。

### 5.6 Claim-evidence 矩阵

| Claim | 状态 | 支持证据 | 反证 | 观察时间 | 下一步 |
|---|---|---|---|---|---|
| 下游需求未来两年增长 |  |  |  |  |  |
| 产品已通过客户认证 |  |  |  |  |  |
| 公司是唯一供应商 |  |  |  |  |  |
| 新产能明年投产 |  |  |  |  |  |
| 合同将产生正 FCF |  |  |  |  |  |
| 当前估值尚未反映 |  |  |  |  |  |

“唯一供应商”是高风险 claim。如果只有公司自己说“领先”，最多标记 `SUPPORTED`；需要客户、竞争者、认证壁垒和替代路径共同验证。

### 5.7 合法、干净的 OSINT 边界

可以使用：

- 公司和监管机构公开页面；
- 公开会议材料和产品文档；
- 合法可访问的专利、招聘、合作伙伴页面；
- 允许引用或摘要的行业资料；
- 公开元数据、链接和原创总结。

不要使用：

- 泄露文件或未授权数据；
- 绕过付费墙、登录或访问控制；
- 私有 API、抓取受限制内容；
- 未经许可复制完整报告、电话会正文或受保护页面；
- 把来源不明截图当作合同证据。

### 5.8 信息冲突怎么处理

当两个来源冲突时：

1. 先比较观察时间，较新资料可能覆盖旧资料。
2. 再比较定义，产能、出货、收入和订单可能不是同一个口径。
3. 检查主体，集团、分部、产品线和单一工厂可能被混用。
4. 检查 GAAP 与 non-GAAP、实际值与预测值。
5. 保留冲突，不强行选一个“看起来顺眼”的数字。
6. 写出冲突会怎样改变 thesis。

> **本章经验（教程操作化）：** 优势来自合法公开信息的拼接，而不是内幕；每个 claim 都要有状态；每个数字都要有时间、质量、许可和 fallback；市场份额必须先定义分母；来源冲突不能被静默覆盖。

> **本章产物：** 为一个关键判断建立 claim-evidence 表，至少包含主张、状态、正证、反证、观察期、来源、下一步七列。

## 6. F — Funnel：先找真实资本流

### 6.1 核心问题

不要先问“哪只 AI 股票会涨”，先问：

1. 最终需求是什么？
2. 谁拥有预算和付款能力？
3. 资本开支已经发生，还是只停留在愿景？
4. 预算流到哪一层？
5. 哪个环节的收入确认最早，哪个最晚？
6. 订单是否会被多家公司重复计算？

### 6.2 从下游往上游追踪

你可以用以下表格记录资本流：

| 层级 | 付款者 | 收款者 | 公开证据 | 时间窗口 | 不确定性 |
|---|---|---|---|---|---|
| 云/模型 | 企业或消费者 | 云厂商 | 收入、合同、使用量 | 当前季度至数年 | 价格、留存 |
| 云 capex | 云厂商 | 服务器/芯片/设备商 | capex 指引、采购承诺 | 1-8 个季度 | 预算延后 |
| 计算平台 | 系统商 | GPU/ASIC/CPU 供应商 | 订单、晶圆、封装 | 2-8 个季度 | 芯片路线变化 |
| 互联和存储 | 系统商 | HBM、网络、光模块商 | 认证、出货、价格 | 2-12 个季度 | 技术替代 |
| 上游材料设备 | 器件厂 | 衬底、设备、材料商 | lead time、扩产、采购 | 3-16 个季度 | 双供、良率 |

> **专业提示｜订单所在层不一定是瓶颈所在层：** 模块厂取得订单，不代表真实约束位于模块层；瓶颈可能继续上移到器件、外延片、衬底、先进封装或生产设备。每一跳都要重新检查产能、良率、认证和替代路径。

### 6.3 资本流的四种证据

从强到弱：

1. 已披露且可约束的合同、采购或资本支出。
2. 公司正式指引、财报附注和管理层明确量产计划。
3. 多个供应链参与者相互印证的订单、价格和产能信号。
4. 行业预测、媒体报道、社交媒体推断。

只看到第 4 类，不能直接得出第 1 类结论。

### 6.4 防止重复计算

假设云厂商宣布 100 个“预算单位”的数据中心建设：

- 服务器厂把 100 单位称为自己的 TAM；
- 芯片厂把其中 50 单位称为可服务市场；
- 网络、存储和电力公司也分别引用同一项目；
- 社交媒体把这些数字相加，误写成 250 单位需求。

正确做法是画资金流，而不是把每一层的 TAM 相加。一个项目的同一美元可以依次成为多家公司的收入，但不能在同一层被重复当作新增终端需求。

### 6.5 Funnel 阶段的交付物

完成一页说明：

```text
最终需求：
预算所有者：
资本开支证据：
传导链：
最可能先确认收入的环节：
最可能后确认收入的环节：
需求可能被取消或延迟的原因：
```

若“谁花钱”答不清楚，停止研究 ticker。

> **专业提示｜预算不是供应商当期收入：** 最终用户、部署方和付款方可能是三类主体；Hyperscaler 的 CapEx 还要经过采购、生产、交付、验收和会计确认，才可能成为某家供应商的收入。

> **本章经验（Serenity 核心）：** 先找付款人，再排证据约束力；明确不同产业层的收入时滞；最后检查是否把同一笔资本开支重复计算。四步中任一步答不清楚，就不要急着找 ticker。

> **本章产物：** 画一张“最终付款人 → 设备/系统 → 器件 → 材料”的资本流图，并给每一跳写出观察指标与时滞。

## 7. A — Architecture：判断架构代际

### 7.1 为什么行业名不够

“光通信受益 AI”太宽泛。你需要知道：

- 当前是可插拔光模块放量，还是 CPO 已经量产？
- 连接是 scale-up 还是 scale-out？
- 光源、DSP、交换芯片、硅光 foundry 各自扮演什么角色？
- 当前产品在下一代还需要，还是会被旁路？

同一主题中的公司，可能分别是当前赢家、过渡赢家、下一代候选和被替代者。

### 7.2 本章只判断架构代际

Architecture 关注的是系统路线怎样变化，而不是再次讲一遍产品商业化流程：

- **当前代际**：已经承担主要部署和收入的路线；
- **过渡代际**：为解决功耗、带宽、成本或兼容性而出现的混合方案；
- **下一代**：可能改变 BOM 和供应商价值量，但尚需商业化验证的路线。

Design win、qualification、pilot、HVM 和 revenue recognition 的证据标准统一放在第 14.1 节。这里先回答“下一代会增加、迁移还是绕开当前部件”，再到第 14 章回答“它是否真的进入商业化”。

### 7.3 架构迁移四问

1. 当前业务到底吃到哪一代收入？
2. 公司迁移到下一代需要新增什么能力？
3. 下一代会扩大、维持还是缩小它的单机价值量？
4. 当前估值反映的是旧业务、本代放量还是下一代成功？

> **历史背景｜出口管制会直接改变架构和可服务市场：** 美国 BIS 于 2022 年 10 月开始实施针对先进计算芯片、超级计算用途和部分半导体制造设备的控制，2023 年以后继续强化和澄清。它不只是一个“宏观风险”标签，还可能迫使产品降规格、重新设计、申请许可证或改变客户与 foundry 路径，因而直接缩小 SAM、延迟收入并改变供应链。来源：[BIS 2022 年规则说明](https://www.bis.gov/press-release/commerce-implements-new-export-controls-advanced-computing-semiconductor-manufacturing-items-peoples)、[BIS 后续强化说明](https://www.bis.gov/press-release/commerce-strengthens-restrictions-advanced-computing-semiconductors-semiconductor-manufacturing-equipment)。

### 7.4 画一张代际表

| 维度 | 当前代际 | 过渡代际 | 下一代 |
|---|---|---|---|
| 技术 |  |  |  |
| 量产状态 |  |  |  |
| 主要客户 |  |  |  |
| 公司产品 |  |  |  |
| 单位价值量 |  |  |  |
| 竞争者 |  |  |  |
| 最大风险 |  |  |  |
| 证据日期 |  |  |  |

如果下一代栏只有 TAM，没有客户、认证和 HVM 时间，就把它标记为“期权价值”，不要当作基准收入。

> **专业提示｜路线图不是收入时间表：** 当前、过渡和下一代技术通常会长期重叠。800G 出现没有让 400G 在同一天消失，CPO 发展也不意味着可插拔光模块立即归零。标准、样品、认证、量产和收入必须分别标记。

> **本章经验（Serenity 核心）：** 始终把当前、过渡和下一代架构分开；商业化阶段不能跳级。只有 TAM、论文和展示的下一代技术先按期权价值处理，而不是直接进入基准收入。

> **本章产物：** 做一张三栏架构表，把当前、过渡、下一代产品分别对应到样品、认证、量产、收入四种状态。

## 8. B — Bottleneck：证明结构瓶颈

### 8.1 好瓶颈的六个条件

| 条件 | 需要证明什么 | 典型反证 |
|---|---|---|
| 需求确定 | 下游真实下单或投入资本 | 预算削减、项目延迟 |
| 扩产困难 | 设备、人才、良率或时间约束 | 竞争者快速扩产 |
| 集中度高 | 少数合格供应商控制供给 | 新进入者很多 |
| 替代困难 | 客户不能短期换方案 | 双供、材料替代、自研 |
| 持续较久 | 错配跨越多个季度 | 一次性补库存 |
| 市场低估 | 证据公开但映射不完整 | 已被一致预期和估值反映 |

### 8.2 短缺不等于瓶颈

三种情况要区分：

| 类型 | 特征 | 投资含义 |
|---|---|---|
| 暂时缺货 | 补库存、物流或单季波动 | 容易快速消失 |
| 周期性短缺 | 供需周期、扩产滞后 | 可能有周期弹性，但要判断峰值 |
| 结构性瓶颈 | 技术、材料、认证、集中度形成长期约束 | 更可能产生持续议价权和重估 |

### 8.3 数量瓶颈与价格瓶颈

一个上游材料即使只占下游 BOM 很小比例，也可能拥有很强议价权。下游不会因为一个占成本很低、但不可替代的部件涨价，就放弃整个高价值 AI 项目。

你需要估算：

```text
组件占下游系统成本比例：
组件停供造成的下游损失：
更换供应商的认证时间：
扩产所需时间：
客户是否可以双供：
价格上涨对客户总成本的影响：
```

### 8.4 五类核心证据

1. **需求证据**：订单、合同、采购、客户 capex。
2. **供给证据**：利用率、产能、扩产周期、设备 lead time。
3. **价格证据**：合同价格、现货价格、管理层涨价口径。
4. **认证证据**：客户 qualification、design win、供应商数量。
5. **交叉证据**：客户、供应商、竞争对手对同一约束的独立确认。

### 8.5 必做的反证测试

每个瓶颈至少写五条解除路径：

```text
竞争者扩产：
现有厂商提高良率：
客户引入第二供应商：
架构或材料替代：
下游需求延迟：
```

若你只能解释为什么会更缺，却解释不出怎样会不缺，你还没有完成研究。

> **本章经验（Serenity 核心）：** 先区分暂时缺货、周期性短缺和结构性瓶颈；再判断停线成本与替换时间，而不是只看 BOM 单价；最后至少写出竞争扩产、良率改善、第二供应商、技术替代和需求延迟五条释放路径。

> **本章产物：** 做一张瓶颈卡：需求证据、供给约束、替代难度、验证指标、五条释放路径和一条最强反证。

## 9. E — Exposure：找到合适的公开市场表达

### 9.1 最纯不等于最好

纯度越高，通常弹性越大，但也可能意味着：

- 单一产品风险更高；
- 客户集中度更高；
- 市值和流动性更小；
- 融资能力更弱；
- 研发或量产失败会更致命。

选择表达的目标不是追求最高纯度，而是比较“纯度 × 证据 × 财务 × 证券结构 × 可交易性”。

### 9.2 六类表达方式

| 表达 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|
| 纯瓶颈小公司 | 主题弹性大 | 二元风险、稀释、流动性 | 证据很强且能承受波动 |
| 多元化大公司 | 财务和流动性较稳 | 相关业务被稀释 | 不确定性高或重视稳定性 |
| 上游材料/设备 | 更接近卡点 | 收入传导较慢 | 认证和扩产约束清楚 |
| 下游集成商 | 订单可见度较高 | 毛利和 capex 压力 | 客户和交付明确 |
| 海外纯标的 | 产业位置可能最精准 | 汇率、披露、治理、交易成本 | 有跨市场研究能力 |
| ETF / basket | 降低单票判断错误 | alpha 被摊薄、成分重复 | 方向明确但赢家不明确 |

### 9.3 纯度要量化

不要只写“公司参与 AI”。至少估算：

```text
相关业务收入 / 总收入：
相关业务毛利 / 总毛利：
未来 capex 中用于该业务的比例：
该业务的客户和订单能见度：
非相关业务可能拖累：
```

如果公司没有分部披露，应把结论标记为估算，并给出上下限，不要伪装成精确事实。

### 9.4 三标的比较法

每个主题至少比较：

1. 最纯的小公司；
2. 流动性更好、但纯度较低的大公司；
3. 海外或上游更纯的替代表达。

比较表：

| 维度 | 标的 A | 标的 B | 标的 C |
|---|---:|---:|---:|
| 相关收入纯度 |  |  |  |
| 相关毛利纯度 |  |  |  |
| 客户质量 |  |  |  |
| 架构位置 |  |  |  |
| 估值 |  |  |  |
| 负债和融资 |  |  |  |
| 稀释 |  |  |  |
| 流动性 |  |  |  |
| 可验证催化 |  |  |  |
| 最大反证 |  |  |  |

找不到合格表达时，“只观察”是有效结论。

> **本章经验（Serenity 核心）：** 最纯的公司不一定是最好的股票表达。尽量量化相关收入和毛利纯度，同时比较小型纯标的、流动性更好的大公司、海外或上游代理，以及“不买、只观察”这一有效选择。

### 9.5 海外表达先做证券身份消歧

| 术语 | 白话解释 | 研究动作 |
|---|---|---|
| 主上市 | 公司股票主要交易和价格发现所在市场 | 优先核对这里的公司公告、股数、币种和交易日 |
| 第二上市 | 同一证券或同一公司在另一交易所正式上市 | 核对是否同股同权、换算比例和可转换机制 |
| OTC 报价 | 场外市场对外国普通股或存托凭证的报价 | 不能仅凭相似代码认定它是主上市或高流动性证券 |
| ADR / ADS | 美国存托凭证安排 / 投资者实际交易的存托股份单位 | 必须记录每份 ADS 代表多少本地普通股、费用和公司行动处理 |
| ISIN | 国际证券识别码 | 用来区分同名公司、不同股类和相似 ticker |

严格说，ADR 常指存托凭证安排或凭证，ADS 是其中代表特定数量本地股票、供投资者交易的单位；市场日常语言可能混用。研究时不要争论简称，而要把 `存托行 + ADS ratio + 对应股类 + 主上市 + 币种` 写清楚。SK hynix 的 `SKHY` 就是后文的实例：1 ADS 代表 0.1 股韩国普通股。

> **行业隐性知识｜代码相同不等于资产相同：** `SIVE` 在 Nasdaq Stockholm 是主上市，`SIVEF` 是美国 OTC 报价；`SOI` 既是 Soitec 的 ticker，也是 silicon-on-insulator 技术缩写。公司名、交易所、股类和 ISIN/ADS 比例必须一起使用。

> **本章产物：** 做一张三标的表达矩阵，并为每个海外标的补齐主上市、股类、代码、ISIN/ADS 比例、币种和可交易性。

## 10. S — Security Structure：先确认股东能否受益

### 10.1 为什么好业务可能是坏股票

一家公司可以同时满足：

- 需求强；
- 客户好；
- backlog 大；
- 收入高速增长；

却仍让普通股股东回报很差，因为建设所需资金来自高息债、持续增发、可转债和巨额 SBC。业务增长属于整个企业，股票回报只属于扣除债权人和新增股东之后的剩余部分。

### 10.2 Fully diluted economics

先把四种口径分开：

```text
期末已发行在外股数：用于点时市值
Basic weighted-average shares：用于 basic EPS
GAAP diluted weighted-average shares：用于 diluted EPS
Scenario fully diluted shares：用于点时稀释压力测试
```

点时稀释情景不是把所有工具的名义股数机械相加。逐项记录：

| 工具 | 需要判断 |
|---|---|
| RSU / PSU / awards | 限制性股票单位 / 绩效股票单位 / 其他奖励的归属、业绩条件、没收、净额结算和实际发行时间 |
| Options / warrants | 行权价、到期日、股价和行权现金；必要时展示 treasury-stock 增量股数与 gross shares 两种情景 |
| Convertibles | 转换条件、现金或股票结算、if-converted 股数、对冲安排 |
| ATM | 剩余额度是融资上限，不是已发行股数；另列实际已售股数和净募资 |

**NCI（Non-controlling interest，非控股权益）**不是股权激励工具，而是合并子公司中不属于母公司股东的那部分权益。存在 NCI 时，子公司的资产、负债和利润可能已并入合并报表，但并非全部经济利益都归上市公司普通股股东。

可转债不能默认只有“全债”或“全股”两种结果，必须按具体结算条款处理：

```text
债务情景：保留债务，不加入转换股数
全额转股情景：加入转换股数，并移除同一笔已假设转换的债务
混合结算情景：保留需现金偿还的本金或现金流出，只计入可能用股票结算的转换溢价股数
```

还要单列 capped call、note hedge 等对冲安排，不能把它们与可转债自动净额抵消。若不按条款选择一致情景，可能同时把同一工具算作债务和股权，或错误删除仍需现金偿还的本金。

教学启发式指标：

```text
情景潜在股本增幅 = (Scenario fully diluted shares / 同时点 current shares outstanding) - 1
ATM overhang proxy = 剩余 ATM 额度 / 同时点当前市值
SBC 强度 = SBC / Revenue
利息覆盖 = EBIT（或 Operating income）/ Cash interest
```

这些指标是本教程为便于操作而增加的，不是原方法论的正式估值公式，也不是统一会计指标。若 EBIT 或 Operating income 小于等于零，利息覆盖应标记为 `N/M`，并直接分析现金利息、到期日、抵押和再融资需求。

两个常见计算方法需要分开：

| 方法 | 白话解释 | 常见用途 |
|---|---|---|
| If-converted method | 假设可转换工具在期初或发行时已经转换，并相应调整利润分子和股数分母 | 可转债 diluted EPS 和情景分析 |
| Treasury-stock method | 假设期权或权证行权所得现金按报告期平均股价回购股票，只计算净新增股数 | 期权、权证等工具的稀释计算 |

> **制度背景｜ASU 2020-06：** FASB 于 2020 年发布 ASU 2020-06，减少部分可转债拆分模型，并要求可转换工具的 diluted EPS 更广泛使用 if-converted method；对多数符合条件的 SEC filer，自 2021 年 12 月 15 日以后开始的财政年度生效。它提高了口径一致性，但 GAAP diluted shares 仍是报告期平均、带反稀释规则的会计数，不等于投资者在当前股价下建立的点时压力情景。来源：[FASB ASU 2020-06](https://storage.fasb.org/ASU%202020-06.pdf)。

> **专业提示｜股数和对冲最容易被算错：** GAAP diluted shares 可能低于高股价或全部转换情景；ATM 授权不等于已经发行；capped call 或 note hedge 可以缓和部分稀释，但不会让债务本金、现金利息和再融资风险消失。

### 10.3 去哪里找证券结构

| 文件 | 重点寻找 |
|---|---|
| 10-K / 10-Q | 债务、SBC、股数、承诺、流动性风险 |
| 8-K | 新合同、融资、重大事件、可转债和增发 |
| S-1 / F-1 | IPO 股本、锁定期、用途、风险、稀释表 |
| 424B prospectus | 发行条款、股数、承销和用途 |
| DEF 14A | 高管薪酬、股权激励、授权股本 |
| Form 4 | 内部人交易；要结合计划和背景，不单独定性 |
| 13D / 13G | 大股东持仓和意图 |

### 10.4 ATM 不是简单的“有或没有”

| 情形 | 处理方式 |
|---|---|
| 额度相对市值很大且持续使用 | 严重降权，必要时剔除 |
| 反复低价发行，同时 SBC 高 | 普通股经济价值高风险 |
| 小比例一次性融资用于明确扩产 | 评估扩产回报和执行 |
| 战略投资者入股并降低债务 | 可能改善资本结构 |
| 为履行高质量合同融资 | 比较合同毛利与融资成本，不能机械判坏 |

ATM 允许公司在额度和条件范围内不时发行，并不意味着全部剩余额度一定会使用。必须分开披露“获准额度、剩余额度、已实际发行股数、平均售价、净募资和用途”。

### 10.5 Neocloud 特别检查

AI 云和数据中心公司至少需要拆：

```text
客户是否有支付能力？
合同现金流是否覆盖运营成本、设备更新/维护 capex、现金利息和债务期限？
GPU 利用率和价格下降会怎样影响毛利？
电力、土地和建设是否已落实？
履约 capex 从哪里来？
利息支出何时超过经营利润？
需要增发多少股才能完成项目？
```

场址、电力和投运状态的完整阶梯见第 2.7 节。这里要把它们放进普通股现金循环：

```text
客户合同和预付款
-> 土地、电力、GPU、网络和建设 CapEx
-> 利息、租赁与设备折旧
-> 利用率和服务收入
-> 维护/更新资本与再融资
-> 留给当前普通股的剩余现金
```

### 10.6 证券结构硬过滤器

以下情况不是自动做空理由，但应阻止它进入初学者核心候选：

- 无法区分期末股数、GAAP diluted EPS 股数和点时稀释情景；
- 融资条款复杂到无法解释；
- 持续经营依赖频繁卖股；
- 客户合同无法覆盖利息和折旧；
- 管理层披露口径反复变化；
- 低 float 和社交热度是主要上涨依据。

> **本章经验（Serenity 核心 + 教程操作化）：** 好业务不等于好普通股；四种股数必须分开；ATM 要看实际使用、价格、用途和资本回报；Neocloud 必须跑完整现金循环；无法解释证券结构时，对初学者应触发 hard gate。

> **本章产物：** 建一张点时资本结构表，分开期末股数、加权平均基本股数、GAAP 稀释股数和 fully diluted 情景，并列出现金、债务、可转债、权证与 ATM。

## 11. R — Repricing：市场为什么会改变看法

### 11.1 催化不是“希望涨”

合格催化必须满足：

1. 有具体事件；
2. 有大致时间；
3. 能改变收入、利润、风险或市场认知；
4. 当前价格尚未完全反映；
5. 事件失败或延迟时有明确处理方式。

### 11.2 常见重估路径

| 催化 | 需要观察 | 可能误读 |
|---|---|---|
| 合同/订单 | 条款、客户、金额、取消权 | 把 MOU 当合同 |
| Qualification | 通过范围、量产平台 | 把认证当收入 |
| Volume ramp | 数量、良率、收入窗口 | 忽略延期和毛利 |
| 涨价 | 合同价、持续时间、客户接受 | 把现货尖峰当长期价格 |
| 财报 | 指引、分部、现金流 | 只看 EPS beat |
| 扩产 | 资本来源、投产时间、回报 | 忽略稀释和过剩 |
| 机构进入 | 13F、持股披露 | 把滞后验证当早期信号 |
| 政策 | 预算、资格、执行细则 | 只买新闻标题 |

### 11.3 5-60 个交易日的正确理解

方法论对历史公开帖子的回看性总结是：部分优质供应链 thesis 曾在 5-60 个交易日后，才通过订单、财报、产业媒体、机构持仓或主流覆盖得到逐步验证。这个区间不是预先注册或独立验证的统计规律。

这不是：

- 上涨保证；
- 目标日期；
- 固定持仓期限；
- 发帖后立即买入的理由。

它只是提醒你：供应链信息转化为市场共识通常有时滞，而不是稳定的一日 copy-trade 信号。

### 11.4 信号分类

| 信号类型 | 研究权重 | 研究处理 |
|---|---:|---|
| New bottleneck thesis | 最高 | 立即验证 BOM、客户、产能、价格和证券结构 |
| Reaffirmation / buy-the-dip | 中高 | 检查基本面是否未变及下跌原因 |
| Supplier map / watchlist | 中 | 建观察清单，等待公司级证据 |
| No-position / exploratory | 低 | 学习过程，不升为核心观点 |
| Victory lap / mainstream validation | 低到中 | 校准旧 thesis，不默认追高 |
| Meme / social-flow | 很低 | 只观察情绪和事件流 |

### 11.5 信息权重不是平权

先区分“作者有多相信”与“公司事实有多强”。

作者观点权重：

```text
文章级完整 thesis > 长篇供应链论证 > 多次一致 reaffirmation > 单条提及
明确研究或持仓 > no-position / exploratory idea
新瓶颈 > supplier map > victory lap
```

基本面证据权重：

```text
监管文件、正式合同和交易对手确认
> 公司第一方指引和产品资料
> 可复核 OSINT 推断
> 社交热度和匿名转述
```

“明确持仓”只能帮助理解 Serenity 的主观 conviction，不能把一个公司 claim 从 `UNKNOWN` 升级为 `VERIFIED`。同时，成熟的 AI 供应链瓶颈 thesis 通常比 squeeze、政治流、meme-flow 和单次事件交易更适合迁移。这些排序都是研究权重，不是正确率保证。

### 11.6 机构滞后

> **制度背景｜13F 为什么存在：** 美国国会在 1975 年建立 Section 13(f) 机构持仓披露制度，目的是提高大型机构证券持仓的公共透明度。达到门槛的投资管理人按季度披露清单内证券，但制度从来不是实时交易广播，也不要求展示完整多空组合。来源：[SEC Form 13F FAQ](https://www.sec.gov/rules-regulations/staff-guidance/division-investment-management-frequently-asked-questions/frequently-asked-questions-about-form-13f)。

常见验证链：

```text
公开供应链证据
-> 公司级认证或订单
-> 行业媒体扩散
-> Sell-side 建模和上调 TAM
-> 机构持仓披露
-> 主流媒体和 victory lap
```

越靠后，证据可能越清楚，但估值也可能越充分。Form 13F 通常最迟在季末后 45 天披露达到 1 亿美元申报门槛的机构管理人持有的 Section 13(f) 清单证券；它不报告股票空头，也不显示建仓日期、成交成本或披露后是否仍持有，部分持仓还可能获准保密处理。某些清单内的持有型 put/call 和权证可能被报告，但 written/short options 不报告。因此，13F 只能用作滞后且不完整的持仓快照，不能把“机构已经买了”当作最早的基本面信息。详见 [SEC Form 13F FAQ](https://www.sec.gov/rules-regulations/staff-guidance/division-investment-management-frequently-asked-questions/frequently-asked-questions-about-form-13f)。

### 11.7 催化日历模板

| 日期/窗口 | 事件 | 会验证什么 | 预期已反映多少 | 失败含义 | 来源 |
|---|---|---|---|---|---|
|  | 财报 | 量产/毛利/订单 |  |  |  |
|  | 行业会议 | 设计和客户 |  |  |  |
|  | 产线投产 | 供给能力 |  |  |  |
|  | 客户平台发布 | 架构需求 |  |  |  |
|  | 解禁/融资 | 股本供给 |  |  |  |

> **本章经验（Serenity 核心）：** 催化剂必须包含事件、时间、状态变化和失败后的处理；5-60 个交易日只是回看性信息扩散区间；社交媒体帖子要先分类；13F 只能作为滞后且不完整的多头持仓快照。

> **本章产物：** 建一张催化日历，每项写明日期窗口、当前状态、预期状态、验证来源和失败后如何降级 thesis。

## 12. P — Portfolio：把观点变成可承受的风险

### 12.1 先区分仓位类型

| 类型 | 研究依据 | 持有逻辑 | 不应混淆为 |
|---|---|---|---|
| Core long | 多年瓶颈、需求和供给证据强 | thesis 持续验证 | 短期财报赌注 |
| Basket | 方向明确、赢家不确定 | 分散单票执行风险 | 真正跨因子分散 |
| Event | 融资、合同、诉讼、上市等事件 | 事件窗口和赔率 | 多年核心仓 |
| Swing / tactical | 情绪、波动和相对滞后 | 明确短期窗口 | 基本面长期投资 |
| Watchlist | 证据不足或价格不合适 | 等待新信息 | “迟早会买” |

零基础课程默认使用普通股观察清单和纸面组合，不需要投入真实资金。事件、低 float、期权和杠杆应留在高级风险教育中；是否使用真实资金是独立且个性化的决定。

### 12.2 Conviction 分层

| 等级 | 定义 | 初学者动作 |
|---|---|---|
| S | 证据完整、反证充分、结构干净、长期瓶颈 | 仍需独立估值和风险预算 |
| A | thesis 清楚，有公司级证据，但有执行或时间风险 | 深度研究候选 |
| B | 方向可能正确，纯度、估值或催化不足 | 观察或篮子候选 |
| C | 供应商地图、探索性想法 | 只做研究 |
| D | 证据弱、结构差或已经拥挤 | 等待重置 |
| F | thesis 被破坏或融资/客户反证明显 | 从候选中剔除 |

等级不是仓位百分比。它只是研究优先级和证据质量标签。

### 12.3 分散 ticker 不等于分散风险

例如以下组合表面有很多股票，但可能仍是单一风险：

```text
多只光器件、衬底和硅光小盘
-> 共同暴露于 CPO / photonics / 高 beta 小盘因子

多只 neocloud 和矿场转数据中心公司
-> 共同暴露于 AI capex、融资利率和 GPU 租赁价格

存储厂、韩国 ETF 和 HBM 设备商
-> 共同暴露于 memory cycle
```

### 12.4 因子暴露表

| 持仓/候选 | AI capex | 利率 | 小盘流动性 | 单一客户 | 中国/出口 | 同一架构 | 同一商品周期 |
|---|---:|---:|---:|---:|---:|---:|---:|
| A | 高 | 中 | 高 | 高 | 低 | 高 | 低 |
| B | 高 | 高 | 高 | 中 | 低 | 中 | 低 |
| C | 中 | 低 | 低 | 低 | 高 | 高 | 中 |

### 12.5 风险预算，而不是“满仓信心”

每个候选都要写：

```text
最坏但合理的业务情景：
最坏但合理的股本情景：
主题整体下跌 30%-40% 时的组合影响：
单一客户丢失时的影响：
我可以承受的最大资金损失：
哪些信息出现后必须重新评估：
```

不要把“我很有信心”当成风险控制。

> **本章经验（Serenity 核心）：** 先定义 core、basket、event、swing 或 watchlist；conviction 只是研究优先级，不是仓位百分比；ticker 数量不是因子分散；风险预算要根据失败情景和最大可承受损失制定。

> **本章产物：** 做一张组合因子暴露表，至少检查 AI CapEx、客户、架构、融资、内存价格、电力和流动性七类共同风险。

## 13. 如何读美股公开文件：SEC、EDGAR 与跨境发行人

### 13.1 从 SEC EDGAR 开始

美国上市公司的核心监管文件可以在 [SEC EDGAR](https://www.sec.gov/search-filings) 查询。SEC 还提供无需 API key 的 [EDGAR submissions 与 XBRL 数据 API](https://www.sec.gov/search-filings/edgar-application-programming-interfaces)，适合核对历史 filing 和标准化财务字段。XBRL 是给财务数据加标准化标签的机器可读格式，但标签一致不保证公司的业务定义完全可比。

初学者不要一上来依赖聚合网站。聚合数据适合筛选，正式结论应回到 filing、附注和公司 IR。

### 13.2 各类文件的用途

| 文件 | 频率/场景 | 研究重点 |
|---|---|---|
| 10-K | 美国公司年度报告 | 业务、风险、三表、附注、客户、债务、SBC |
| 10-Q | 美国公司季度报告 | 最新经营、现金流、股本和风险变化 |
| 8-K | 重大事项 | 财报新闻稿、需披露的重大最终协议、融资、管理层变动、资产交易；并非所有商业合同都会出现，部分条款可依法隐去 |
| S-1 | 美国 IPO 注册文件 | 商业模式、股本、用途、锁定期、风险和稀释 |
| 424B | 发行说明书 | 最终发行条款、价格、股数、承销和用途 |
| DEF 14A | 股东大会委托书 | 高管薪酬、治理、激励计划、关联交易 |
| Form 4 | 内部人交易 | 买卖、授予、行权；需要结合交易计划解释 |
| 13D / 13G | 大股东披露 | 持股比例、主动或被动意图 |
| 13F | 达到 1 亿美元门槛的机构管理人季末 13(f) 清单持仓 | 通常最迟季后 45 天；不含股票空头和 written options，无成本和建仓时间，可能有保密处理 |
| 20-F | 外国发行人年度报告 | 类似 10-K，注意 IFRS/披露差异 |
| 6-K | 外国发行人临时报告 | 财报、公告和重大事项 |

SEC 也提供了面向投资者的 [10-K 阅读指南](https://www.sec.gov/investor/pubs/reada10k.pdf) 和 [8-K 阅读指南](https://www.sec.gov/files/readan8k.pdf)。

外国发行人还要注意会计与时间边界：

| 概念 | 初学者提醒 |
|---|---|
| US GAAP | 美国会计准则，常见于美国发行人 |
| IFRS / K-IFRS | 国际财务报告准则及韩国采用版本；现金流分类等细节可能与 US GAAP 不同 |
| 财政年度 | 公司自行定义的报告年度；FY2027 Q1 不一定发生在自然年 2027 年第一季度 |
| Pro forma | 假设出售、收购或重组已经完成后的重列口径，不等于当期实际 GAAP 报告值 |

### 13.3 读 10-K 的顺序

第一次读一家公司，建议按以下顺序，而不是从第一页读到最后一页：

1. `Business`：公司卖什么、客户是谁、怎么赚钱。
2. `Risk Factors`：客户集中、供应链、技术、融资和监管风险。
3. `MD&A`：管理层解释收入、毛利、费用和现金变化。
4. 财务三表：利润表、资产负债表、现金流量表。
5. 收入分拆和客户集中附注。
6. 债务、可转债和流动性附注。
7. SBC、股本和每股收益附注。
8. Commitments and contingencies：采购、租赁、诉讼和承诺。

### 13.4 财报电话会的十个问题

1. 哪个产品真正驱动本季增长？
2. AI 相关收入是披露值、管理层估算还是市场推断？
3. 客户是新增、扩容还是提前拉货？
4. 订单、backlog 和收入确认如何对应？
5. 量产、良率和交付时间是否改变？
6. 涨价来自真实供需还是产品组合？
7. 毛利改善来自价格、成本、利用率还是会计口径？
8. 为增长需要多少 capex、库存和营运资金？
9. 融资、SBC 和股数如何变化？
10. 哪一句话能证伪原 thesis？

### 13.5 管理层语言翻译表

| 管理层表述 | 你应追问 |
|---|---|
| “Strong demand” | 订单金额、客户数、期限和取消权？ |
| “Engaged with customers” | 样品、design win、认证还是量产？ |
| “Strategic partnership” | 有约束合同和现金流吗？ |
| “Massive TAM” | 公司能拿多少份额，何时产生收入？ |
| “Adjusted profitability” | 排除了哪些成本？GAAP 和 FCF 怎样？ |
| “Fully funded” | 资金来自现金、债务、可转债还是 ATM？ |
| “Sold out” | 什么产品、到何时、价格和毛利是否锁定？ |

> **本章经验（教程操作化）：** 聚合网站适合筛选，正式结论必须回到 filing、附注、公司 IR 和交易对手披露；读取跨境公司时，还要同时核对法定实体、交易所、会计准则、币种和财政年度。

> **本章产物：** 用最新一份正式 filing 完成一次十问阅读清单，并把每个答案链接到原始文件的具体章节或附注。

## 14. 商业化与合同：从样品、认证到收入和现金

读完监管文件后，下一步不是马上估值，而是把“技术进展”和“合同新闻”翻译成可以进入财务模型的事实。商业化是一条连续链，任何一步都不能跳过：

先把本章反复使用的合同词放在同一张卡上：

| 词 | 白话解释 | 不能自动证明什么 |
|---|---|---|
| LOI / MOU | 意向书 / 谅解备忘录，通常表达合作方向 | 已签不可取消订单 |
| RFQ | Request for Quotation，客户询价流程 | 已中标、已量产或已有收入 |
| HVM | High-Volume Manufacturing，大批量制造 | 高良率、高毛利或现金已经到账 |
| Backlog | 公司定义的待履约订单或合同池 | 全部不可取消、全部会确认收入 |
| RPO | 会计规则下尚未确认的履约义务 | 当季收入、利润或现金 |
| TCV | Total Contract Value，合同总价值 | 一年收入；它可能跨多年并附条件 |
| ARR | Annual Recurring Revenue，年化经常性收入 | GAAP 收入；硬件或一次性合同不能随意年化成 ARR |
| Take-or-pay | 客户承诺最低采购/支付的合同机制 | 无条件现金；仍要看最低量、抵扣、终止和信用风险 |

```text
研究 / 原型
-> 样品
-> Design win
-> Qualification
-> Pilot
-> RFQ / Allocation
-> Signed contract
-> HVM / Volume ramp
-> 发货或提供服务
-> Revenue recognition
-> 收款
-> 扣除履约资本后的自由现金流
```

### 14.1 每一个商业化阶段能证明什么

| 阶段 | 它能证明什么 | 仍然不能证明什么 |
|---|---|---|
| 原型 / 展示 | 技术在受控条件下可工作 | 客户愿意采购 |
| Design win | 产品进入特定客户设计 | 已通过全部认证或形成收入 |
| Qualification | 通过特定平台、产品或工厂认证 | 所有客户通用、永久独供 |
| Pilot | 小批量验证工艺和良率 | 稳定产能、稳定毛利 |
| RFQ / Allocation | 采购流程或产能安排推进 | 已签约、不可取消 |
| Signed contract | 存在正式合同关系 | 全部金额都会确认收入 |
| HVM / Volume ramp | 进入规模量产或爬坡 | 高良率、高毛利和永久份额 |
| Revenue recognition | 履约义务已经满足并确认收入 | 现金已经全部到账或资本回报良好 |

> **经验法则｜商业化阶段不能跳级：** 下一代技术只有 TAM、论文或展示时，应先视为期权价值；只有认证、订单、量产、收入和现金流依次得到验证后，才能逐步进入 base case。

### 14.2 从合同到收入：先过 Topic 606 五步法

> **制度背景｜为什么会有 Topic 606：** FASB 与 IASB 在 2002 年启动统一收入确认项目，2014 年发布趋同准则；美国公众公司主要从 2018 年开始采用，其他实体主要从 2020 年开始采用。它把大量行业专用规则收束为以合同、履约义务和控制权转移为核心的五步模型。来源：[FASB Topic 606 实施后评估](https://storage.fasb.org/Post-Implementation%20Review%E2%80%94Revenue%20from%20Contracts%20with%20Customers%20%28Topic%20606%29.pdf)。

Topic 606 的核心不是记会计术语，而是强迫研究者回答五个问题：

1. 是否存在符合条件的客户合同？
2. 合同中有几项独立履约义务？
3. 交易价格是多少，是否包含可变对价？
4. 交易价格怎样分配给各项履约义务？
5. 公司是在某一时点还是一段时间内履约并确认收入？

因此：

```text
签约 ≠ 收入
预付款 ≠ 毛利
Backlog ≠ RPO
TCV ≠ 一年收入
ARR ≠ GAAP Revenue
Take-or-pay ≠ 无条件现金
```

第 3.3 节只负责把读者导航到这里；本章才是这些合同术语、收入确认与合同经济性的完整解释位置。

### 14.3 合同证据按约束力排序

| 证据 | 通常权重 | 必须继续检查 |
|---|---:|---|
| 产品展示、合作讨论 | 很低 | 是否只是营销或探索 |
| LOI / MOU | 低 | 是否有约束力、期限和退出条款 |
| RFQ / Allocation | 中 | 是否有数量、价格和客户承诺 |
| Signed contract | 中高 | 取消权、履约义务、客户信用 |
| Take-or-pay | 高于普通意向 | 最低用量、抵扣、终止、退款和可收回性 |
| 已交付并确认收入 | 高 | 毛利、收款和履约资本 |
| 已收现金且产生正 FCF | 最强经济验证 | 是否可持续、是否依赖一次性预付款 |

### 14.4 合同进入模型前的定性门

本节只回答“这份合同能否进入财务模型”，不在这里重复做定量表。依次检查：

1. 谁是合同主体和最终付款人，客户信用是否足够；
2. 合同是否有约束力，是否允许取消、退款、抵扣或重谈；
3. 收入在什么条件和期间确认；
4. 履约需要哪些设备、电力、库存、租赁和营运资本；
5. 毛利能否覆盖折旧、利息与潜在稀释。

五项通过后，再到第 15.5 节量化未来收入、GAAP 毛利、CapEx、利息、稀释和普通股剩余价值。

> **行业隐性知识｜大合同也可能伤害普通股：** 如果履约需要先投入巨额设备、电力和建设资金，而合同毛利不足以覆盖折旧、利息和融资稀释，那么 headline backlog 越大，资金缺口反而可能越大。

### 14.5 四个最容易误判的合同指标

1. **Backlog、ARR、TCV 和 RPO 不可互换。** 它们的定义、时间范围、取消权和会计基础不同。
2. **Take-or-pay 不是无条件收入。** 仍要看最低用量、履约条件、终止、抵扣和客户信用。
3. **预付款不等于利润。** 它可能只是在履约前改善现金，同时形成合同负债和未来资本需求。
4. **HVM 不是商业化终点。** 量产之后仍要验证良率、产品组合、价格、毛利、收款和份额持续性。

### 14.6 本章 Hard Gate

以下任一项答不清楚，合同不能直接进入估值模型：

- 合同主体和付款人是谁；
- 是否具有约束力，是否可以取消或重谈；
- 收入确认与收款分别发生在什么时候；
- 毛利和履约 CapEx 是多少；
- 需要债务、租赁还是增发才能完成；
- 如果客户延迟、价格下降或利用率不足，谁承担损失。

> **本章产物：** 把一条商业新闻标成“宣传、样品、design win、认证、合同、HVM、收入、收款”中的准确状态，并完成合同 Hard Gate。

## 15. 财务质量：从故事走到股东回报

### 15.1 收入桥

最基础的收入模型：

```text
Revenue = Units shipped × Average selling price
```

从市场推导时：

```text
Company revenue
= Total market units
× Company market share
× Average selling price
× Revenue recognition factor
```

其中每一个变量都需要范围，而不是单点幻想。

### 15.2 毛利桥

```text
Gross profit = Revenue × GAAP gross margin
简化 Operating profit
= Gross profit
- R&D
- Sales & marketing
- G&A
- 其他营业费用
```

实际报表还可能包含重组、减值、并购摊销和其他营业项目。利息通常位于营业利润以下；SBC 和折旧可能分布在销售成本与各类 Opex 中。因此，GAAP gross margin 只是其中一层，不能代表完整经济成本。

毛利改善可能来自：

- 价格上涨；
- 高毛利产品占比增加；
- 产能利用率提高；
- 良率改善；
- 物流和材料成本下降；
- 会计重分类或调整口径。

只有前五项可能改善真实经济性；第六项可能只是展示方式变化。

### 15.3 GAAP 与 non-GAAP 对账

建一张桥接表：

| 项目 | GAAP | 调整项 | Non-GAAP | 你是否接受调整 |
|---|---:|---:|---:|---|
| Gross profit |  |  |  |  |
| Operating income |  |  |  |  |
| Net income |  |  |  |  |
| EPS |  |  |  |  |

常见调整：SBC、重组、折旧摊销、并购费用、减值和一次性诉讼。不要机械否定所有调整，但必须判断它是否真的一次性。如果每年都“调整掉”，它就是经济成本的一部分。公司提供 non-GAAP 指标时，应同时查看最可比的 GAAP 指标、对账表和定义是否变化；可参考 [SEC Non-GAAP Financial Measures 指引](https://www.sec.gov/rules-regulations/staff-guidance/corporation-finance-interpretations/non-gaap-financial-measures)。

> **制度背景｜Regulation G：** 互联网泡沫和重大会计事件之后，SEC 在《萨班斯-奥克斯利法案》背景下于 2003 年实施 Regulation G，要求公开 non-GAAP 指标不得具有误导性，并同时提供最可比的 GAAP 指标及定量对账。它没有禁止调整后指标，但确立了“补充而不能替代 GAAP”的边界。来源：[SEC Regulation G 最终规则](https://www.sec.gov/files/rules/final/33-8176.htm)。

> **专业提示｜调整后利润不能越过经济成本：** 如果 SBC、折旧、重组或融资费用多年重复发生，它们就不是因为被称作“调整项”而自动消失。至少沿 `Gross profit -> Operating income -> Net income -> CFO` 走完一次，再判断调整是否合理。

### 15.4 现金流和资本强度

高增长不一定创造自由现金流。FCF 没有统一 GAAP 定义；以下只是常见教学近似：

```text
简化 FCF = CFO - Capex
```

研究时必须记录公司自己的 FCF 定义并与 GAAP 现金流对账。`CFO - capex` 也不一定等于可以自由分配的现金：它可能遗漏融资租赁新增、资本化软件、强制偿债或其他必要支出。在 US GAAP 下现金利息通常进入 CFO；IFRS 的分类选择可能不同。

估值分母必须一致：

```text
Levered / equity FCF -> 对比 Equity value / Market cap
Unlevered FCF -> 对比 Enterprise value
```

CFO 会加回非现金 SBC，因此分析 FCF 时还要同时检查净股数变化和为抵消稀释所消耗的回购现金。

进一步问：

- 库存是否吃掉现金？
- 应收账款是否增长快于收入？
- 客户是否预付款？
- 扩产 capex 是维护性还是增长性？
- 设备多久折旧，技术多久淘汰？
- 现金缺口用债务还是股权填？

> **专业提示｜预付款不等于毛利已经实现：** 客户预付款可能改善短期 CFO，但公司仍可能承担未来服务、GPU、电力和建设义务，并在资产负债表形成合同负债。必须把预付款与履约 CapEx、收入确认和退款/抵扣条款同时分析。

### 15.5 合同经济学进入财务模型

第 14 章已经判断合同处于哪个商业化阶段；本节只做下一步：把合同转换为利润、现金和普通股剩余价值。不要只抄 headline backlog，逐项填写：

| 项目 | 内容 |
|---|---|
| 合同状态 | LOI / MOU / signed / take-or-pay |
| 总金额 |  |
| 合同期限 |  |
| 年化收入 |  |
| 取消/重谈条款 |  |
| 客户信用 |  |
| 收入确认窗口 |  |
| 预计 GAAP 毛利 |  |
| 履约 capex |  |
| 资金来源 |  |
| 利息和稀释成本 |  |
| 对普通股的剩余价值 |  |

教学启发式筛选：

```text
Forward contracted revenue / equity value
= 可合理归属于未来 12 个月的约束性合同收入 / 同时点股权价值

Forward contracted revenue / EV
= 可合理归属于未来 12 个月的约束性合同收入 / 同时点企业价值
```

这两个比率都不是标准指标，也不是偿债覆盖率。分子必须披露合同约束性、取消权、确认概率和时间口径；一次性硬件 backlog 的年化收入不能随意称为 SaaS ARR。它们只能帮助发现待研究的错配，不能替代期限、毛利、capex、债务、稀释和估值。高收入比率但负毛利、高 capex 或巨额稀释，仍然可能毁灭股东价值。

### 15.6 客户集中度的双面性

高质量大型客户可以验证：

- 需求真实；
- 技术通过严格筛选；
- 付款能力更强；
- 合同更可能融资。

但也带来：

- 议价权集中；
- 丢单冲击；
- 定制成本；
- 单一平台延迟；
- 客户自研和双供风险。

所以“Mag7 客户”是需求质量过滤器，不是利润保证。

> **本章经验（Serenity 核心 + 会计操作化）：** 毛利改善必须沿完整利润链核对；收入增长不等于 FCF；合同要扣除履约资本和融资成本；Mag7 客户提高需求质量，但不保证利润或续约。

> **本章产物：** 做一张收入—毛利—营业利润—CFO—CapEx—FCF 桥，并解释增长最终是否转成每股自由现金流。

## 16. 估值：好公司是否已经太贵

### 16.1 三个不同问题

每次决策都分开回答：

1. **业务 thesis 是否成立？**
2. **当前价格隐含了什么？**
3. **即使业务成立，未来回报是否足够补偿风险？**

不要用“公司会增长”直接回答第三问。

### 16.2 基础估值公式

```text
Market cap = Share price × Current shares outstanding

Scenario diluted equity value
= Share price × Scenario fully diluted shares

Enterprise value
= Market cap
+ Debt
+ Preferred equity
+ Minority interest
- Cash and equivalents

EV / Sales = Enterprise value / Revenue
P / E = Market cap / Net income attributable to common shareholders
Equity FCF yield = Levered / equity FCF / Market cap
```

其中 **Preferred equity** 是在分配顺序上通常优先于普通股的优先股权，**Minority interest** 是合并报表中不属于母公司股东的少数股东权益；两者都不能因为不在普通股市值中就从企业价值分析里消失。

这些是简化公式。股价、市值、现金、债务和股数等点时项目应使用同一估值日；Revenue、Net income、FCF 等期间项目则分别标注 TTM、财年、季度或 forward 期间。对每一项可转换证券，要按实际条款选择债务、全额转股或现金本金加股票溢价的混合结算情景；不能把已假设转股的同一部分同时留在 debt 中，也不能删除仍需现金偿还的本金。期权和权证的行权现金也应通过净增股数或现金调整体现。

不同数据平台可能对现金、租赁负债、养老金、非经营投资、可转债和情景 diluted shares 处理不同，研究报告必须写清口径。上述 `FCF yield` 只适用于一致定义的 equity/levered FCF；unlevered FCF 应与 EV 配对。

P/E 的利润分母必须明确是 TTM、正常化还是 forward，并使用归属于普通股股东的同一期间口径；亏损、利润接近零或一次性项目主导时，P/E 通常不具可比意义。

### 16.3 Pre-ramp 公司怎么估

当前收入很小，不代表可以跳过估值。应使用情景法：

| 变量 | Bear | Base | Bull |
|---|---:|---:|---:|
| 量产时间 |  |  |  |
| 客户数量 |  |  |  |
| 份额 |  |  |  |
| ASP |  |  |  |
| 收入 |  |  |  |
| GAAP 毛利率 |  |  |  |
| Opex |  |  |  |
| Capex |  |  |  |
| 新增股数 |  |  |  |
| 合理倍数 |  |  |  |
| 概率 |  |  |  |

再做概率加权：

```text
Probability-weighted value
= Bear value × Bear probability
+ Base value × Base probability
+ Bull value × Bull probability
```

Bear、Base、Bull 概率之和必须等于 100%，三种价值必须使用相同计价单位和估值时点。概率不是事实；你必须解释每个概率来自哪些里程碑、失败率或可比案例。

### 16.4 反向估值

与其问“能值多少”，不如问当前价格要求公司做到什么：

```text
当前企业价值：
市场隐含未来收入：
市场隐含毛利率：
市场隐含份额：
市场隐含量产时间：
要实现这些假设，公司必须通过哪些里程碑？
```

如果当前价格已经隐含 bull case，业务做对也不一定有好回报。

### 16.5 TAM 的正确用法

```text
TAM
-> 可服务市场 SAM
-> 公司实际可竞争市场
-> 可获得份额
-> 单位价格
-> 收入
-> 毛利
-> 经营利润
-> 自由现金流
-> 每股价值
```

任何一步缺失，都不能从“大 TAM”直接跳到“股价应翻倍”。

SAM 不是 TAM 的固定比例。产品规格、出口管制、认证范围、地域、客户类型和产能都可能缩小公司真正可服务的市场。尤其在先进计算和半导体设备领域，政策变化可能直接改变可销售产品和客户集合，因此必须把出口限制放进 bear/base/bull 情景，而不是只写在风险因素末尾。

> **本章经验（教程操作化）：** 业务 thesis、当前价格和未来预期回报是三个问题；pre-ramp 公司也必须估值；先从当前价格反推市场隐含的份额、毛利和量产时间，再决定赔率是否足够。

> **本章产物：** 完成 Bear、Base、Bull 三情景和一次反向估值，明确当前价格隐含的收入、份额、毛利率与时间。

## 17. 评分与 Hard Gate：排序工具，不是买入机器

评分的用途是启发式排序研究优先级，不是机械生成买卖或仓位。这张表没有经过统计校准，分数不代表成功概率、预期回报、最大回撤或安全仓位。

| 模块 | 满分 | 评分问题 |
|---|---:|---|
| 资本流确定性 | 10 | 下游 capex 是否明确，客户是否真实花钱？ |
| 瓶颈强度 | 15 | 供给是否难扩、难替代、有议价权？ |
| BOM / OSINT 证据 | 10 | 能否画清多跳供应链并交叉验证？ |
| 架构位置 | 10 | 当前、过渡和下一代角色是否明确？ |
| 暴露纯度 | 10 | 收入、毛利和估值是否真正暴露于瓶颈？ |
| 合同经济学 | 10 | 合同、客户、收入和履约成本是否清楚？ |
| 财务质量 | 10 | GAAP、SBC、折旧、利息和 FCF 是否支持？ |
| 证券结构 | 10 | 债务、ATM、可转债、权证和股本是否可控？ |
| 重估催化 | 8 | 有可验证时间窗口和事件吗？ |
| 仓位适配 | 7 | 与风险预算和组合因子是否匹配？ |

### 17.1 分数解释

| 总分 | 方法论原始含义 | 初学者更保守的用法 |
|---:|---|---|
| 80-100 | 核心候选，继续深挖 | 进入最高优先级复核，不自动买入 |
| 65-79 | 篮子或中等候选 | 深挖弱项，等待证据或价格 |
| 50-64 | 观察、事件或小仓 | 只放观察清单 |
| 0-49 | 暂不交易 | 停止，除非出现结构性新信息 |

### 17.2 防止“高分幻觉”

评分前执行三个规则：

1. `UNKNOWN` 不得按中性分；关键模块未知时先不计算总分，若筛选流程必须给分则保守记零并保留 `UNKNOWN` 标签。
2. 单一来源不能同时为多个模块加满分。
3. 一票否决项优先于总分。

例如，一家公司可能因为巨大 TAM、强客户和高纯度获得较高分，但如果 ATM 额度很大且公司持续发行，证券结构可以触发 hard gate。反过来，只有剩余额度而没有实际发行，也不能把额度直接当成已发生稀释。

> **本章经验（教程操作化）：** 评分只决定研究优先级，不产生买入指令。关键项 `UNKNOWN`、强反证和 hard gate 永远优先于总分。

> **本章产物：** 填一张带 `UNKNOWN` 的评分卡，并在总分旁单列 hard gate、最强反证和仍需补证的三项问题。

## 18. 标准研究流程：从一个线索到一份报告

> **执行原则（教程操作化）：** 用时间门保护研究效率：先做 30 分钟筛选，再做 90 分钟 BOM 映射；只有通过后才进入 filing、财务桥、估值和同行比较。买入前先写反方论点，退出条件和更新节奏必须在行动前完成。

### 18.1 第一道：30 分钟快速筛选

目标不是决定买入，而是决定是否值得花更多时间。

```text
[ ] 公司卖什么？
[ ] 在 AI capex 哪一层？
[ ] 客户是谁？
[ ] 收入规模和市值大致多少？
[ ] 最近一份 10-K/10-Q/20-F 是何时？
[ ] 是否有明显债务、ATM、可转债或持续经营风险？
[ ] 线索来自事实、推断还是社交帖子？
[ ] 最大反证是什么？
```

如果产品、客户和股本三项都说不清，停止。

### 18.2 第二道：90 分钟产业链定位

1. 画出客户到上游的五层 BOM。
2. 标记公司产品所在节点。
3. 写当前、过渡和下一代架构。
4. 找至少一个客户和一个供应商/竞争者交叉验证。
5. 写出瓶颈解除路径。

交付物是一张链图和一页 claim-evidence 表。

### 18.3 第三道：半天第一轮公司深挖

按顺序读：

1. 最新 10-K/20-F；
2. 最新 10-Q/6-K；
3. 最近重大 8-K；
4. 最近两个季度财报材料；
5. 债务、股本、SBC、合同和客户附注；
6. 公司与交易对手产品资料；
7. 可信行业资料。

完成财务桥、股本桥和合同经济学表。

这只是第一轮初筛，不表示一家复杂供应链公司已经达到真实资金决策所需的研究深度。重大合同、跨境证券、复杂可转债或技术路线争议可能需要多轮核验和专业意见。

### 18.4 第四道：同行比较

至少找两个替代表达：

- 更纯但风险更高；
- 更稳但纯度更低；
- 若可能，再加海外纯标的或 ETF。

若你只研究了一家公司，很容易把公司宣传误当行业事实。

### 18.5 第五道：写反方报告

这一步也叫 **Red team**：站到最强反方的位置，主动攻击自己的因果链，而不是找一个容易反驳的稻草人。

强迫自己写一页：

```text
为什么需求可能被高估？
为什么瓶颈可能解除？
为什么公司拿不到份额？
为什么收入不能变成利润？
为什么融资会吞掉回报？
为什么当前价格已经太贵？
为什么催化会延迟？
```

### 18.6 第六道：预先写退出条件

分开三类：

| 类型 | 含义 | 示例 |
|---|---|---|
| Thesis broken | 核心事实失效 | 丢失认证、客户取消、替代出现 |
| Valuation stretched | 业务仍好但回报不足 | 价格已隐含极端份额和毛利 |
| Better expression | 找到更优工具 | 更纯、更便宜或结构更干净的同行 |

### 18.7 第七道：建立更新节奏

| 频率 | 更新内容 |
|---|---|
| 每周 | 关键新闻、filing、价格/产能证据、claim 状态 |
| 每季 | 财务模型、股数、债务、指引、催化和评分 |
| 重大事件 | 合同、融资、产品延迟、客户变更、政策 |
| 价格大变动 | 先查事实，再判断是估值变化还是 thesis 变化 |

回到第 1 章的三道门：产业真实受益、公司获得收入利润现金、价值留给当前普通股。只有三道门连续通过，下面七个问题才有讨论意义。

### 18.8 七个必须回答的问题

```text
谁花钱？
卡在哪里？
谁控制卡点？
为什么现在没被定价？
什么时候会被定价？
股东结构有没有坑？
错了怎么知道？
```

如果答不清楚，这个 idea 还不是合格的核心研究对象。

### 18.9 标准结论句式

```text
我研究这家公司，不是因为它被归类为 AI 概念股，
而是因为资本开支正在从 [下游需求] 传导到 [具体环节]；
该环节受 [材料 / 产能 / 良率 / 认证 / 设备 / 工程能力] 约束；
[公司] 是公开市场中在 [纯度 / 流动性 / 估值] 之间较合适的表达；
其合同、GAAP 财务和点时稀释/证券结构 [支持 / 不支持] 普通股回报；
未来 [具体事件与时间] 可能让市场重新定价；
如果 [具体反证] 出现，thesis 将被降级或失效。
```

### 18.10 三种正确结论

研究不只会得到“买”或“卖”：

1. **继续深挖**：产业、公司和结构都值得进一步验证。
2. **观察等待**：方向可能对，但证据、价格或证券结构尚不合格。
3. **明确放弃**：无法验证、股东经济性差或存在更好表达。

对初学者，第二和第三种结论往往比仓促交易更有价值。

### 18.11 最后检查

```text
[ ] 这不是旧推文或历史案例被误当成当前信号
[ ] 当前价格、财务、合同和股本已重新核验
[ ] 每个核心 claim 都有来源、日期和状态
[ ] 架构层级没有混淆
[ ] 瓶颈有需求、供给和替代三方面证据
[ ] 合同不是 MOU/LOI 冒充
[ ] GAAP、SBC、折旧、利息和 FCF 已检查
[ ] 期末股数、GAAP diluted EPS 股数、点时稀释情景、ATM、可转债和权证已检查
[ ] 催化有事件、时间和失败含义
[ ] Thesis broken 与 valuation stretched 已分开
[ ] 组合按风险因子而非 ticker 数量评估
[ ] 当前动作允许是“什么都不做”
```

> **本章产物：** 写出一页标准研究备忘录：一句话 thesis、证据状态、财务与股本桥、估值、催化、反证、组合风险和当前动作。

## 19. 完整虚构案例：从 AI 集群追到 InP 上游

> 本章所有公司、合同、数字和事件均为教学虚构，不对应任何真实证券。货币统一使用“单位”，目的只是演示研究过程。

### 19.1 原始线索

一则假设新闻称：大型云厂商 `Orion Cloud` 计划未来两年扩大 AI 集群，下一代交换平台需要更多高速光连接。

社交媒体结论是：

> “光模块要爆发，所有光通信股票都会涨。”

你的任务不是接受这句话，而是把它改写成可验证假设：

```text
Orion 的资本开支是否已经批准？
新增集群需要多少高速端口？
端口从当前代际升级到什么速率？
铜与光的比例如何变化？
光模块中的哪一个部件最可能受限？
限制来自模块、激光器、外延片还是衬底？
谁拥有合格产能？
```

### 19.2 三家虚构候选

| 公司 | 角色 | 优点 | 初始风险 |
|---|---|---|---|
| ModuleMax | 大型光模块和系统制造商 | 流动性好、客户多、交付能力强 | 相关业务只占一部分，纯度低 |
| LaserPure | 高速激光器纯标的 | AI 相关收入纯度高、进入两个设计 | ATM、权证、负 FCF |
| InPSource | InP 衬底供应商 | 上游纯度高、认证慢、供应商少 | 客户集中、海外流动性较差 |

### 19.3 Step F：确认资本流

假设你找到以下材料：

| 证据 | 来源等级 | 状态 | 能证明 | 不能证明 |
|---|---|---|---|---|
| Orion 在监管文件中披露上调 capex 指引 | L1 | `VERIFIED` | 公司确实发布了这项指引 | 资本开支已实际发生、最终金额或具体供应商 |
| Orion 网络负责人介绍高速互联升级 | L2 | `SUPPORTED` | 架构方向 | 每端口用量和时间 |
| ModuleMax 称客户询价上升 | L2 | `SUPPORTED` | 需求线索 | 已签订单 |
| 社媒称 LaserPure 是独家供应商 | L5 | `UNKNOWN` | 提供调查线索 | 任何独家关系 |

结论：资本流方向得到支持，但未来支出仍是指引，公司归属也未确认。不能从云 capex 直接跳到 LaserPure。

### 19.4 Step A：拆架构代际

| 维度 | 当前 | 过渡 | 下一代 |
|---|---|---|---|
| 连接 | 可插拔高速光模块 | 更高集成度光引擎 | CPO / 硅光深度集成 |
| LaserPure | 当前产品已有小批量 | 新激光器在认证 | 能否适配新光源架构未知 |
| InPSource | 衬底供应现有器件 | 新规格需客户再认证 | 若替代材料成熟，需求可能改变 |
| 主要证据 | 收入与出货 | design win / qualification | 样机、客户路线图 |

结论：本代需求较明确，下一代仍是期权价值。估值时不能把下一代全部计入基准情景。

### 19.5 Step B：验证瓶颈

假设调查结果如下：

```text
需求：高速端口量预计增加，但具体数量仍是管理层预测。
扩产：InPSource 新晶体产线需要 18-24 个月建设和认证。
集中度：当前有三家可供货，两家通过最高规格认证。
替代：客户正在测试第二供应商和另一种器件路线。
价格：长协价格稳定，尚无公开涨价证据。
库存：模块端库存正常，没有明显抢货。
```

这说明：

- 上游存在潜在约束；
- 但“当前已经严重短缺”证据不足；
- 第二供应商和替代路线是关键反证；
- 最合理状态是 `SUPPORTED`，不是 `VERIFIED`。

### 19.6 Step E：比较暴露

假设财务拆分：

| 指标 | ModuleMax | LaserPure | InPSource |
|---|---:|---:|---:|
| AI 光互联相关收入占比 | 25% | 75% | 85% |
| 相关毛利贡献估算 | 35% | 80% | 90% |
| 客户数量 | 多 | 2 个主要项目 | 3 个主要客户 |
| 业务纯度 | 低到中 | 高 | 很高 |
| 流动性 | 高 | 中 | 低 |
| 海外/汇率风险 | 低 | 低 | 高 |

如果只看纯度，InPSource 最强；如果考虑流动性、披露和客户集中，答案不再简单。

### 19.7 Step S：LaserPure 的股本桥

假设 `m` 代表百万股。以下是研究者建立的点时情景，不是 GAAP diluted EPS：

```text
Current shares outstanding = 100m（1 亿股）
Unvested RSUs gross = 5m；按归属情景计入 4m
Warrants gross = 10m；按行权价和当前股价估算净增 6m
Convertible if-converted shares = 20m；本情景假设转换并移除同一笔债务
Scenario fully diluted shares = 100m + 4m + 6m + 20m = 130m
Remaining ATM capacity = 40 units
Current market cap = 200 units
```

计算：

```text
情景潜在股本增幅 = (130 / 100) - 1 = 30%
ATM overhang proxy = 40 / 200 = 20%
```

`30%` 是特定转换和行权假设下的压力测试，不是已发生或必然发生的稀释；`20%` 只是剩余融资额度相对市值的 overhang proxy，也不是预计发行比例。即使业务判断正确，股东仍可能面对显著稀释。你必须进一步核对：

- 权证是否全部价内；
- 可转债转换价格和对冲安排；
- ATM 是否正在使用；
- 融资用于高回报扩产，还是填补持续亏损；
- 扩产后每股自由现金流能否增加。

这个步骤展示了“好业务不等于好股票”。

### 19.8 Step R：催化和反证

| 窗口 | 可能催化 | 升级条件 | 失败信号 |
|---|---|---|---|
| 30-60 天 | 客户完成认证 | 客户或双方正式确认 | 认证延迟 |
| 下一财季 | 收入和毛利开始改善 | 量与 GAAP 毛利同时提升 | 只增收入不增现金 |
| 6-12 个月 | 新产线投产 | 良率和客户订单到位 | capex 超支、稀释扩大 |
| 12-24 个月 | 下一代平台导入 | 产品进入 HVM | 架构绕开现有产品 |

### 19.9 Step P：评分与 hard gate

| 模块 | ModuleMax | LaserPure | InPSource | 说明 |
|---|---|---|---|---|
| 资本流确定性 /10 | 7 | 7 | 7 | 有正式指引，但不是已发生支出 |
| 瓶颈强度 /15 | 3 | 5 | 8 | 潜在上游约束；短缺、涨价和独家地位未验证 |
| BOM 证据 /10 | 6 | 6 | 6 | 链条合理，但每跳仍需外部交叉验证 |
| 架构位置 /10 | 6 | 6 | 6 | 下一代替代风险仍在 |
| 暴露纯度 /10 | 5 | 9 | 10 | 使用给定的虚构分部估算 |
| 合同经济学 /10 | `UNKNOWN` | `UNKNOWN` | `UNKNOWN` | 没有合同条款，不得制造分数 |
| 财务质量 /10 | `UNKNOWN` | 3 | `UNKNOWN` | 只有 LaserPure 负 FCF 线索，仍不完整 |
| 证券结构 /10 | `UNKNOWN` | 2 | `UNKNOWN` | 只有 LaserPure 建了情景股本桥 |
| 重估催化 /8 | 4 | 5 | 5 | 认证和量产都可能延迟 |
| 仓位适配 /7 | `UNKNOWN` | `UNKNOWN` | `UNKNOWN` | 未完成个人风险和组合分析 |
| 总分 /100 | `NOT SCORED` | `NOT SCORED` | `NOT SCORED` | Hard gate：关键模块未知，禁止汇总 |

### 19.10 为什么不能机械按分数买

- ModuleMax 可以用于同行和行业验证，但合同、财务和证券结构尚未填完。
- LaserPure 的纯度较高，但点时稀释情景显示显著风险，需核验实际转换、行权和 ATM 使用；合同经济学与仓位适配等关键模块仍为 `UNKNOWN`，因此禁止汇总总分。
- InPSource 的上游位置提高了研究价值，却不能替代短缺、涨价、客户和财务证据。
- `UNKNOWN` 不是中性分，也不能为了得到总分而填入主观数字。

一个合格的阶段性结论：

> 资本流和光互联升级方向得到支持，上游 InP 约束值得继续研究，但当前短缺和独家地位尚未被交叉确认。三家公司都缺少完成股票排名所需的合同、财务或证券结构数据；LaserPure 还出现明显稀释情景风险。因此当前只能升级主题研究，不能生成公司总分或买入结论。

### 19.11 案例的 thesis card

```text
Thesis：AI 集群升级将提高高速光器件需求，上游合格 InP 供给可能先于下游形成约束。

已验证：Orion 披露上调 capex 指引。
得到支持：本代高速网络升级方向。
待验证：InP 实际短缺；供应商份额；下一代仍使用当前材料。
关键反证：第二供应商快速通过认证；替代架构减少 InP 用量；下游项目延迟。
证券风险：LaserPure ATM/权证/可转债；InPSource 低流动性和客户集中。
```

为什么不能把四个数字相加成“AI CapEx”？

- Microsoft 的披露包含资产取得与租赁时点影响，并另披露现金支付 PPE；
- Alphabet 使用现金流量表中的 purchases of property and equipment；
- Amazon 披露现金购置、资产出售激励、融资租赁新增资产等多组口径；

- 四家公司都不会把全部 CapEx 逐笔标成 AI、非 AI、土地、建筑、电力、网络或服务器。

因此，正确结论是“资本流仍然强，且多个买家披露容量约束”，不是“把四个数相加后全部分配给 GPU”。

三种 backlog 的边界不同：

电力链还要分清三个词：**Behind-the-meter** 是用户电表后的现场供电或发电安排；**Dry-type transformer** 是以空气等方式绝缘和散热的干式变压器；**Oil-filled transformer** 是以绝缘油冷却和绝缘的油浸式变压器。它们属于不同产品和应用分母，不能用一家公司在某个窄市场的份额代表整个数据中心配电市场。

场址、电力、设备和收入状态的完整定义见第 2.7 节。本节只把那条投运链应用到真实公司：`land/site secured`、`power contracted`、`energized`、`IT installed`、`commissioned` 和 `revenue-generating` 之间没有跨公司的统一自动换算，`active power` 更需要回到公司定义。

### 20.9 第一组实战：SNDK、MU、Samsung、SK hynix 的存储比较

**公司历史｜2016 年收购，2025 年重新拆分。** Western Digital 原本以 HDD 为核心，2016 年完成对 SanDisk 的收购，以进入 NAND、SSD 和移动存储；其年度报告披露交易总对价约 US$15.59B。两类业务在同一集团运营多年后，WDC 于 2025 年 2 月完成 Flash 业务分拆，独立 Sandisk 在 Nasdaq 以 `SNDK` 交易，WDC 此后聚焦 HDD。来源：[Western Digital 2016 年报](https://investor.wdc.com/static-files/65bbf704-73ac-4e81-9139-2b774229d66a)、[Sandisk 分拆完成公告](https://investor.sandisk.com/news-releases/news-release-details/sandisk-celebrates-nasdaq-listing-after-completing-separation)、[Western Digital 分拆后年度报告](https://investor.wdc.com/static-files/06884eb3-7909-470f-a6a3-9d2f2275b4af)。

> **行业隐性知识｜历史可比性先于增长率：** 分拆前后的收入、利润、债务、现金和股数不能机械连续比较，旧研究中的“WDC=NAND/Flash”映射也已经失效。研究存储周期前，先确认今天买的是 HDD、NAND/SSD，还是 HBM/DRAM 资产。

**本组先学会八个词：**

| 词 | 白话解释 | 使用边界 |
|---|---|---|
| ASP | Average Selling Price，平均售价 | 价格上涨能推高收入，但不能替代真实出货增长 |
| Bit shipment / exabytes | 分别是按存储 bit 和 EB 计量的容量出货 | 它们衡量容量，不是设备台数，也不是收入 |
| eSSD | Enterprise SSD，企业级固态硬盘 | 受服务器和数据中心需求驱动，但不等于纯 AI 收入 |
| NBM | Sandisk 对 New Business Model 多年客户安排的公司称谓 | 公司称其由明确财务承诺支持；它不是通用会计术语，仍要逐份看期限、取消权和收入确认 |
| JV | Joint Venture，合资企业；本组主要指 Sandisk 与 Kioxia 的闪存制造合作 | JV 资本开支、晶圆成本和现金流不一定全部出现在合并报表同一行 |
| RPO | Remaining Performance Obligations，剩余履约义务 | 是未来可能确认收入的合同义务，不是当季收入、利润或现金 |
| SOCAMM2 | 面向 AI 服务器的低功耗 LPDDR5X 内存模组 | 它补充服务器内存层级，不能与 HBM 收入或份额相加 |
| M15X / P&T7 | SK hynix 的 DRAM 晶圆厂 / AI 内存先进封装厂 | 建厂、装机、认证、量产和形成合格收入是不同状态 |

补充一个更早期概念：**HBF（High Bandwidth Flash）**是用 NAND 面向大容量 AI 推理内存需求的下一代架构探索，截至本快照仍应按研发与标准化阶段处理，不能把它写进当前基准收入。

四家公司都出现了异常高的毛利和利润，但不要把它理解成四家都形成永久垄断。存储利润大致可拆成：


### 20.10 第二组实战：LITE、COHR、AAOI 的光互联比较

第 2.6.1 节已经说明 400G、800G、1.6T 与 CPO 的标准演进。这里不再把技术史单列成附录，而是直接用三家公司验证它的投资含义：可插拔模块仍在产生当前收入，1.6T 正在爬坡，CPO、外置光源和 OCS 是后续状态；任何一家公司都不能因为展示了下一代产品，就跳过认证、HVM、收入和现金转换。

**本组先学会八个词：**

| 词 | 白话解释 | 在投资判断中代表什么 |
|---|---|---|
| EML | Electro-absorption Modulated Laser，电吸收调制激光器 | 高速光模块的一类关键发射器件；出货增长仍需落到收入与良率 |
| DFB | Distributed Feedback Laser，分布反馈激光器 | 可作连续光源或通信激光；规格、功率和应用不同，不能只数颗数 |
| CW laser | Continuous-Wave Laser，连续波激光器 | 在硅光和 CPO 中提供持续光源，可能由外置光源系统集中供光 |

| LRO | Linear Receive Optical，线性接收光学方案 | 用更简化的接收链降低功耗，但仍处过渡与验证阶段 |
| NPO | Near-Packaged Optics，近封装光学 | 光学靠近交换芯片，但没有像 CPO 那样与芯片共同封装 |
| ELS | External Laser Source，外置激光源 | 把激光器移到封装外，便于供光、散热和维护；合同、可靠性与认证仍要验证 |

### 20.11 第三组实战：AXTI、SIVE、SOI 的上游材料与激光比较

#### 20.11.1 先分清 substrate、laser 与 engineered substrate

AXTI、SIVE 和 SOI 都能出现在“AI 光子上游”叙事里，但并不处于同一 BOM 层：AXTI 生产 InP 等**块状衬底**，SIVE 设计/制造 InP **激光器与阵列**并经营 Wireless，SOI 用 Smart Cut 等工艺生产 Photonics-SOI 等**工程化衬底**。三者没有一个可以用另外两家的收入倍数直接替代。

**本组先学会九个词：**

| 词 | 白话解释 | 使用边界 |
|---|---|---|
| Bulk substrate | 块状衬底，晶体本身构成晶圆主体；AXTI 属于这一层 | 衬底收入不能直接等同下游激光器或模块收入 |
| Engineered substrate | 通过键合、薄层转移等方式组合材料特性的工程化衬底；SOI 属于这一层 | 它与块状 InP 衬底不是同一产品和分母 |
| Smart Cut | Soitec 的晶圆键合与薄层转移工艺平台 | 工艺 IP 是壁垒之一，但仍要看利用率、认证与产品组合 |
| Photonics-SOI | 面向硅光器件的绝缘体上硅衬底 | 可服务数据中心光互联，也不能把全部销售都标成 AI |
| RF-SOI / FD-SOI | 面向射频 / 全耗尽数字芯片的 SOI 平台 | 与 Photonics-SOI 共用公司平台，但终端需求和周期不同 |
| Imager-SOI / POI / Power-SOI | 面向图像传感 / 压电材料 / 功率器件应用的工程化衬底 | 这些收入解释了为什么 SOI 的公司收入并非纯数据中心 AI |
| NRE | Non-Recurring Engineering，一次性工程开发收入 | 能证明项目活动，不代表稳定产品收入或量产份额 |
| PON | Passive Optical Network，无源光网络 | AXTI 的 InP 需求也可能来自宽带接入，不能全归因于 AI 数据中心 |
| Current operating result | Soitec 的经调整经常性经营口径 | 不是完整 IFRS 营业利润，必须与减值和其他经营费用对账 |

### 20.12 十家公司横向归纳：方法论真正改变了什么

| 研究问题 | 存储四家给出的答案 | 光互联三家给出的答案 | 上游三家给出的答案 |
|---|---|---|---|
| 哪个数字最能证明当前收入？ | 可报告 Memory/DRAM/NAND/Data Center 收入 | D&C/Data Center/OCS 等分部或产品收入，但要剥离非 AI | SOI 的 Photonics-SOI 最直接；AXTI/SIVE 的纯 AI 收入仍 unknown |
| 最大架构误判 | 把 HBM 份额与 DRAM 份额相加，或把 NAND 当 HBM | 把 800G、1.6T、LRO、OCS、CPO 当一次性切换 | 把 substrate、engineered substrate、laser 当同一产品 |
| 最大瓶颈误判 | 高份额不自动等于持续短缺 | 展示/产能不等于良率和认证后的量产 | 安装产能不等于出口许可、客户资格和可交付产能 |
| 最大收入纯度误判 | Samsung DS、MU 全部收入或 SNDK 全部收入都不是纯 AI | LITE Components、COHR D&C、AAOI DC 均不是精确 AI 收入 | AXTI substrate、SIVE Photonics、SOI Edge & Cloud AI 都含非目标工作负载 |
| 最大证券结构风险 | 周期峰值利润、SKHY ADS、普通/优先股、股数分母 | 可转债、优先股、ATM、权证、客户持股 | 大额增发、债转股、NCI、会计重述和 pro forma 现金 |
| 催化如何写 | “HBM4E 从样品到量产”“RPO 转收入” | “800G/CPO 从认证到收入并转成现金” | “6-inch/新加坡产能从安装到合格出货” |
| 反证如何写 | ASP、份额、CapEx 回报和库存反转 | 良率、客户集中、CFO、库存和继续融资 | 许可、认证、合作停滞、现金消耗和稀释 |

十家公司没有一个可以只靠“AI TAM 很大”完成 thesis。最成熟的 S 层公司也可能处在利润率高点；最上游、最稀缺的公司也可能因许可、良率或融资无法把价值留给普通股股东；技术路线正确也可能因为市场已经按数十倍销售额定价而缺乏安全边际。

把研究结论压缩成一句标准句式：

```text
因为 [下游资本流] 正在推动 [明确架构变化]，
[具体产品/能力] 可能成为 [有证据的瓶颈]；
[公司] 通过 [可核验收入/合同/产能状态] 获得 [多少暴露]，
但当前普通股还要承担 [债务/稀释/估值/客户/周期]；
下一次可验证状态变化是 [日期或报告期 + 指标]，
若 [反证] 发生，thesis 失效。
```

举三个不同成熟度的示例：

- **MU：** HBM4 已量产且数据中心收入可量化，但高毛利与高 CapEx 都处在极端区间；下一步验证 HBM4E、份额、ASP 和 FCF，若长协无法抵御价格反转或 CapEx 回报下降，不能再按当前利润外推。
- **AAOI：** 800G 已首次批量交付，Data Center 收入也已增长，但公司仍亏损、CFO 为负并经历 ATM；下一步必须看到收入转毛利和现金，若产能只增长不产生合格出货，thesis 失败。
- **SIVE：** 多个合作仍处开发/qualification，Photonics 收入反而下降，连续融资后估值主要押注未来；下一步不是再收集合作 logo，而是等量产收入、毛利和现金消耗改善，若继续融资却不转量产，应下调 thesis 阶段。

### 20.13 三组案例的七层差异：只做最后一次对照

七层定义已经在第 6—12 章学过，这里不再复述。现在只比较同一问题在三组公司中为什么会得到不同答案：

| 层 | 存储四家 | 光互联三家 | 上游三家 |
|---|---|---|---|
| F：资本流 | AI 服务器带动 HBM、DRAM、NAND 与企业 SSD，但同一服务器需求会跨产品重复出现 | Hyperscaler 网络升级传到可插拔、激光器、OCS 与 CPO | 下游扩产再传到 InP、激光阵列和工程化衬底，收入时滞最长 |
| A：架构 | HBM4/4E、SOCAMM2、eSSD 和 HBF 处于不同阶段 | 800G 已量产，1.6T 爬坡，LRO/NPO/CPO/ELS 处于不同阶段 | 块状衬底、激光器、Photonics-SOI 不是一条可互换技术路线 |
| B：瓶颈 | 要看 ASP、bit shipment、良率、封装与扩产，不只看份额 | 要看 InP 供给、器件良率、可靠性、认证和模块测试 | 要看晶体质量、出口许可、工艺 IP、客户认证和合格产能 |
| E：公开表达 | 可报告收入较多，但产品周期和业务混合度不同 | Data Center 分部是代理值，不是精确 AI 收入 | 纯度看似更高，实际可核验 AI 收入往往更少 |
| S：证券结构 | 双市场 ADS、普通/优先股和周期峰值利润最关键 | 可转债、ATM、权证、客户持股与高 CapEx 最关键 | 增发、债转股、NCI、重述与流动性最关键 |
| R：状态变化 | RPO 转收入、HBM4E 量产、份额与 ASP | 800G/1.6T 量产转毛利和现金，CPO 从展示转认证 | 许可、6-inch、新加坡产线从安装转合格出货 |
| P：共同因子 | 内存价格、GPU/ASIC 出货和 hyperscaler CapEx | hyperscaler CapEx、InP、客户认证和光通信估值 | 光互联周期、出口政策、融资和小盘流动性 |

> **行业隐性知识｜越上游，验证时滞往往越长：** 产业链位置更“稀缺”不等于股价更安全。上游公司可能最早获得询单，却更晚形成合格出货；若期间需要持续融资，产业价值可能先被债权人或新股东拿走。

## 24. 高级模块：理解即可，不急着使用

### 24.1 期权和 IV 错价

Serenity 方法论包含 Vega、IV、LEAPS、covered calls 和 ETF 波动率错价，但它们是独立的高级策略栈。

| 术语 | 最简含义 | 初学者首先要知道的风险 |
|---|---|---|
| Call / Put | 在约定条件下买入 / 卖出标的的权利 | 买方可能损失全部权利金 |
| Option writer | 期权卖方承担被指派后的履约义务 | 某些卖方策略损失可能超过初始收入 |
| Strike / Premium | 行权价 / 权利金 | 方向正确不保证覆盖价格和时间成本 |
| IV / Vega | 隐含波动率 / 期权对 IV 变化的敏感度 | IV 下跌可能抵消标的方向收益 |
| Theta / Delta | 时间衰减 / 对标的价格小幅变化的敏感度 | 时间和敏感度都在变化，不是固定杠杆 |
| LEAPS | 较长期限的期权 | 期限长不等于风险低 |
| Covered call | 持股同时卖出看涨期权 | 限制上涨，但仍承担标的大部分下跌风险 |
| Bid-ask spread | 买价与卖价之差 | 价差大意味着很高的隐性交易成本 |

使用期权前必须能回答：

```text
方向 thesis 是什么？
催化预计何时发生？
到期日是否覆盖延迟？
当前 IV 相对什么基准高或低？
Vega、Theta 和 Delta 如何影响结果？
方向正确但 IV 下跌时会怎样？
最大损失是多少？
流动性和 bid-ask spread 如何？
```

[Investor.gov 的期权入门公告](https://www.investor.gov/introduction-investing/general-resources/news-alerts/alerts-bulletins/investor-bulletins-63)提醒，期权买方可能损失全部权利金，某些卖方策略的潜在损失可能更高；未覆盖看涨期权卖方的损失理论上可能没有上限。对初学者，最合理的处理通常是先完成普通股研究和纸面练习，而不是用期权放大尚未成熟的判断。

### 24.2 高 short interest 和 squeeze

```text
高 short interest + 盈利增长 + 近端催化
```

可能形成反身性，但：

```text
高 short interest + 亏损 + 融资压力 + 低流动性
```

更可能是风险警报。Short interest 是结构变量，不是方向信号。

### 24.3 海外纯标的

海外市场可能存在更纯的材料、设备和供应链公司，但额外风险包括：

ADR、ADS、主上市和 OTC 的身份区别已在第 9.5 节解释；本节只讨论把海外证券用于真实投资时新增的风险。

- 披露语言、频率和会计准则；
- 汇率；
- 交易时间和流动性；
- 治理和少数股东保护；
- ADR 比例、托管费用和退市安排；
- 税务、市场准入和结算。

[Investor.gov 的国际投资说明](https://www.investor.gov/introduction-investing/investing-basics/investment-products/international-investing)也强调，外国公司提供的信息可能不同、未必使用英语，国际投资还涉及额外成本和汇率等风险。

### 24.4 ETF 和篮子

ETF 能降低单一公司风险，但窄行业 ETF 可能仍高度集中。检查：

- 前十大持仓；
- 同一主题权重；
- 指数规则和再平衡；
- 费用和流动性；
- 是否持有你原本想回避的低质量公司；
- 多个 ETF 是否重复持仓。

[Investor.gov 的资产配置与分散说明](https://www.investor.gov/introduction-investing/getting-started/asset-allocation)指出，窄行业基金本身不一定提供充分分散，投资者仍需检查实际持仓是否重复。

## 25. 结语：把研究变成可重复习惯

Serenity 方法论最值得学习的，不是某一个 ticker，也不是某一次公开交易，而是一种可重复的研究习惯：

```text
从资本流开始，
用架构判断方向，
用 BOM 找到卡点，
用公开证据验证，
用财务和证券结构过滤，
用催化解释时间，
用组合和退出条件管理错误。
```

真正成熟的研究不是“我找到了会涨的股票”，而是：

> 我知道自己相信什么、证据是什么、哪些仍是推断、什么会证明我错，以及即使我错了也不会失去继续研究的能力。
